# Internationalization (i18n)

## Overview

The site supports English (`en`) and Chinese (`zh`) using [next-intl](https://next-intl-docs.vercel.app/).

## Configuration

### Locale Config (`i18n/config.ts`)

```typescript
export const locales = ['en', 'zh'] as const;
export const defaultLocale = 'en';
export type Locale = (typeof locales)[number];
```

- **Locale prefix:** `always` â€” every URL starts with `/en/` or `/zh/`
- **Default locale:** `en` (root `/` redirects to `/en`)

### Request Handler (`i18n/request.ts`)

Uses `getRequestConfig` from `next-intl/server` to load messages dynamically:

```typescript
const messages = (await import(`../messages/${locale}.json`)).default;
```

## URL Structure

```
/en/                    # English homepage
/zh/                    # Chinese homepage
/en/projects/           # English projects
/zh/projects/           # Chinese projects
/en/services/kitchen/   # English kitchen service
/zh/services/kitchen/   # Chinese kitchen service
```

## Translation Files

Located in `messages/`:

| File | Language | Keys |
|------|----------|------|
| `en.json` | English | Navigation, CTAs, section headings, form labels, meta, footer |
| `zh.json` | Chinese | Same keys, Chinese translations |

Key namespaces: `nav`, `cta`, `hero`, `section`, `label`, `footer`, `benefits`, `services`, `projects`, `blog`, `contact`, `stats`, `metadata`, `serviceBenefits`, `locationBenefits`, `areaBenefits`

### Usage in Components

```typescript
// Server component
import { useTranslations } from 'next-intl';

export default function MyComponent() {
  const t = useTranslations('Navigation');
  return <nav>{t('home')}</nav>;
}
```

## Bilingual Data Model

Content uses the `Localized<T>` type, whether stored in static files (`lib/data/`) or fetched from the database (`lib/db/queries.ts`):

```typescript
type Localized<T> = { en: T; zh: T };

// Example: project title

```

Helper functions extract the right locale:

```typescript
function getLocalizedProject(project: Project, locale: Locale) {
  return {
    title: project.title[locale],
    description: project.description[locale],
    // ...
  };
}
```

## Database Columns

The DB schema uses `*En` / `*Zh` suffix convention:

```typescript
titleEn: varchar('title_en', { length: 256 }),
titleZh: varchar('title_zh', { length: 256 }),
```

## Navigation

`navigation.ts` exports locale-aware navigation utilities:

```typescript
import { Link, redirect, usePathname, useRouter } from '@/navigation';
```

These automatically include the current locale prefix in URLs.

## Locale Switching

The `Navbar` component includes a locale switcher that preserves the current path when switching between English and Chinese.
