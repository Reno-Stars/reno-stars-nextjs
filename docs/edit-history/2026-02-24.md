**Session 10** ŌĆö Fix new project display order. New projects were inserted with `displayOrderInSite: 0` (the default), causing them to appear in an unpredictable position among existing projects that already occupied position 0. Fixed by querying `max(display_order_in_site)` for the site and placing new projects at the end.

### Changes Made

1. **New project ordering fix**: `createProject()` now queries the current max `displayOrderInSite` for the target site and sets the new project's order to `max + 1`. Uses `coalesce(max(...), -1)` so the first project in an empty site gets order `0`.
