**Session 10** — Fix new project display order. New projects were inserted with `displayOrderInSite: 0` (the default), causing them to appear in an unpredictable position among existing projects that already occupied position 0. Fixed by querying `max(display_order_in_site)` for the site and placing new projects at the end.

### Changes Made

1. **New project ordering fix**: `createProject()` now queries the current max `displayOrderInSite` for the target site and sets the new project's order to `max + 1`. Uses `coalesce(max(...), -1)` so the first project in an empty site gets order `0`.

**Session 11** — Cascade site deletion. Previously, deleting a site with projects was blocked (`onDelete: 'restrict'`). Changed to cascade deletion so deleting a site removes all child projects automatically. ConfirmDialog now lists project names before confirming. Removed unused `cannotDeleteWithProjects` translation key.

### Changes Made

1. **Schema FK change**: `projects.siteId` FK changed from `onDelete: 'restrict'` to `onDelete: 'cascade'`. Full cascade chain: site → projects → (image pairs, scopes, external products), plus site → site image pairs. Blog posts with `projectId` reference use `onDelete: 'set null'` (unaffected).
2. **Server action simplified**: `deleteSite()` no longer checks project count or blocks deletion — just deletes the site row and lets DB cascade handle cleanup. Removed unused `projects` and `count` imports.
3. **ConfirmDialog items**: Site delete dialog now lists all project names (localized by admin locale) via the `items` prop, so users see exactly what will be deleted.
4. **Translation updates**: `deleteMessage` updated to "This will permanently delete the site and all projects inside it." Removed unused `cannotDeleteWithProjects` key from both EN and ZH.

**Session 12** — Add PO Number field to projects and sites. Internal purchase order number for sales staff to quickly find projects. Editable in admin forms, searchable in admin sites list and public projects page, but not visually displayed to customers.

### Changes Made

1. **Database schema**: Added `poNumber: varchar('po_number', { length: 50 })` to both `projectSites` and `projects` tables in `lib/db/schema.ts`.
2. **Server actions**: Added `poNumber` extraction via `getString(formData, 'poNumber') || null` to `getSiteData()` in `app/actions/admin/sites.ts` and `getProjectData()` in `app/actions/admin/projects.ts`.
3. **Type system**: Added `po_number?: string` to `Site`, `Project`, and `LocalizedProject` interfaces in `lib/types.ts`. `DisplayProject` inherits from `LocalizedProject` automatically.
4. **Query layer**: Added `po_number` mapping in `mapDbProjectToProject()` and `mapDbSiteToSite()` in `lib/db/queries.ts`. Added `poNumber` to `ProjectSummary` interface and `getAllProjectsBySiteAdmin()` select.
5. **Localization**: Added `po_number` passthrough in `getLocalizedProject()` in `lib/data/projects.ts`.
6. **Admin forms**: Added `poNumber` to `initialData` interface and a `FormField` after the slug field in both `ProjectForm.tsx` and `SiteForm.tsx`. Added `poNumber` to `ProjectWithDetails` and `SiteData` interfaces in `SiteDetailClient.tsx`, plus the `getProjectFormData()` mapping. Added `poNumber` to site data mapping in the edit page `app/admin/(dashboard)/sites/[id]/page.tsx`.
7. **Admin sites list**: Added `poNumber` to `SiteRow` and `StandaloneProjectRow` interfaces. Added PO Number column first in both Sites and Standalone tabs. Added `poNumber` to search filters for sites, nested projects, and standalone projects.
8. **Public projects page**: Added `searchQuery` state with text input in filters section. Extended `filteredProjects` memo to match on `po_number`. Passed `po_number` through site→DisplayProject conversion. Updated `clearFilters` and `hasActiveFilters` to include search.
