# Edit History: 2026-02-24

## Summary

**Session 9** — Admin dashboard completeness + about page landing preview. Added a landing page preview to the About admin page (matching the card grid pattern used by FAQs, Services, etc.), and added a Settings section to the admin dashboard with link cards for the 3 singleton pages (Company, Showroom, About) that were previously missing. Followed by a code review that identified 5 issues — all fixed: unified duplicate card components, extracted shared styles, fixed indentation, and made the preview grid responsive.

### Changes Made

1. **About landing page preview**: Added a 3-column responsive card grid below the `AboutForm` that mirrors the homepage `AboutSection` layout — gold divider bar, bold title, 3-line clamped text. Uses `useAdminLocale` for EN/ZH switching. Conditionally rendered when at least one section has content.
2. **Dashboard Settings section**: Added a 4th section ("Settings") to the admin dashboard with link-only cards (no count number) for Company, Showroom, and About — the 3 singleton admin pages that were previously missing from the dashboard.
3. **Unified `DashboardCard` component**: Merged the separate `SettingsLinkCard` into `DashboardCard` by making `value` optional. When `value` is undefined, renders a compact row layout (icon + label); when present, renders the original column layout with count.
4. **Extracted dashboard styles**: Created `sectionHeaderStyle` and `sectionGridStyle` constants to eliminate duplicated inline style objects across the sections loop and the (former) Settings block.
5. **Responsive about preview grid**: Changed from fixed `repeat(3, 1fr)` to `repeat(auto-fill, minmax(220px, 1fr))` so preview cards stack on narrow screens.
6. **Fixed AboutForm indentation**: Fragment children (`<form>` and preview section) now consistently indented at 6 spaces from the return statement.

**Session 10** — Fix new project display order. New projects were inserted with `displayOrderInSite: 0` (the default), causing them to appear in an unpredictable position among existing projects that already occupied position 0. Fixed by querying `max(display_order_in_site)` for the site and placing new projects at the end.

### Changes Made

1. **New project ordering fix**: `createProject()` now queries the current max `displayOrderInSite` for the target site and sets the new project's order to `max + 1`. Uses `coalesce(max(...), -1)` so the first project in an empty site gets order `0`.

---

**Session 11** — Cascade site deletion. Previously, deleting a site with projects was blocked (`onDelete: 'restrict'`). Changed to cascade deletion so deleting a site removes all child projects automatically. ConfirmDialog now lists project names before confirming. Removed unused `cannotDeleteWithProjects` translation key.

### Changes Made

1. **Schema FK change**: `projects.siteId` FK changed from `onDelete: 'restrict'` to `onDelete: 'cascade'`. Full cascade chain: site → projects → (image pairs, scopes, external products), plus site → site image pairs. Blog posts with `projectId` reference use `onDelete: 'set null'` (unaffected).
2. **Server action simplified**: `deleteSite()` no longer checks project count or blocks deletion — just deletes the site row and lets DB cascade handle cleanup. Removed unused `projects` and `count` imports.
3. **ConfirmDialog items**: Site delete dialog now lists all project names (localized by admin locale) via the `items` prop, so users see exactly what will be deleted.
4. **Translation updates**: `deleteMessage` updated to "This will permanently delete the site and all projects inside it." Removed unused `cannotDeleteWithProjects` key from both EN and ZH.
