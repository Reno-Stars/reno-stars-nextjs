## Session 2: Process Page â€” Export Centering Fix, Layout Improvements, Code Review Cleanup

Fixed poster PNG/PDF export centering issue, improved step card layout and reveal timing, extracted SVG illustrations to separate file, and applied two rounds of code review fixes. Files changed: `components/pages/ProcessPage.tsx`, `components/illustrations/process.tsx` (new), `messages/en.json`, `messages/zh.json`.

### Changes

#### 1. Export centering fix â€” width-locking for html-to-image
- **File:** `components/pages/ProcessPage.tsx`
- Root cause: `html-to-image` renders in SVG `foreignObject` where block elements don't inherit full parent width, breaking `margin: auto` centering
- Fix: Before `toPng` capture, lock all container elements' widths to computed pixel values via `querySelectorAll('section, [class*="max-w-"], [class*="flex-col"], [class*="flex-row"]')`
- After capture, restore original widths in `finally` block
- Cards container changed from `space-y-*` to `flex flex-col items-center gap-*` with `w-full` children

#### 2. Viewport-based card reveal (replaced road Y-position thresholds)
- **File:** `components/pages/ProcessPage.tsx`
- Cards previously appeared based on abstract SVG road Y-positions (`STEP_ROAD_Y_POSITIONS`), causing early steps to pop up before users could see them
- Now uses each card's actual `getBoundingClientRect().top` compared to `windowH * CARD_REVEAL_VIEWPORT` (0.85)
- Removed `STEP_REVEAL_OFFSET`, `STEP_ROAD_Y_POSITIONS`, and `STEP_CIRCLES` constants

#### 3. Step number circles â€” HTML elements replacing SVG
- **File:** `components/pages/ProcessPage.tsx`
- Replaced SVG-based step number circles (fixed viewBox coordinates) with HTML `<div>` elements positioned at bottom-center of each card row
- SVG circles couldn't align with HTML flow layout; HTML elements use `absolute -bottom-5 left-1/2 -translate-x-1/2`

#### 4. Alternating zigzag card layout
- **File:** `components/pages/ProcessPage.tsx`
- Cards alternate `lg:flex-row` / `lg:flex-row-reverse` for visual dynamism
- Illustration column: `lg:w-[22%]` with directional alignment (`lg:justify-end` / `lg:justify-start`)

#### 5. SVG illustrations extracted to separate file
- **New file:** `components/illustrations/process.tsx`

- Also exports `stepIllustrations` array

#### 6. Discriminated union for trust badges and footer stats
