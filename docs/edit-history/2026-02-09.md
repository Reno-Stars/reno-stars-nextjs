## SearchableSelect Component

Created a new reusable component at `components/admin/SearchableSelect.tsx` for searchable dropdown selections in admin forms.

### Features

- **Type-to-filter search** — Filter options by typing in the input field
- **Keyboard navigation** — Arrow keys to navigate, Enter to select, Escape to close
- **Click outside to close** — Uses `useRef` and `mousedown` event listener
- **Clear button** — X button to clear selection (only visible when value selected and not disabled)
- **Visual checkmark** — Gold checkmark icon for currently selected option
- **Hidden input** — For form submission via standard FormData
- **Scroll into view** — Highlighted option scrolls into view in dropdown
- **Disabled state** — Respects `disabled` prop with `readOnlyStyle`

### Props Interface

```typescript
interface SearchableSelectProps {
  name: string;                    // Form field name
  options: SearchableSelectOption[]; // { id: string; label: string }[]
  value: string;                   // Currently selected option ID
  onChange: (value: string) => void; // Selection change handler
  placeholder?: string;            // Input placeholder text
  noResultsText?: string;          // Text when no options match search
  clearText?: string;              // Aria label for clear button
  disabled?: boolean;              // Disable interaction
}
```

### Accessibility

- `role="combobox"` on input with `aria-expanded`, `aria-haspopup="listbox"`, `aria-autocomplete="list"`
- `role="listbox"` on dropdown list
- `role="option"` on each option with `aria-selected`
- `aria-label` on clear button

### Styling

- Uses `inputStyle`/`readOnlyStyle` from `shared-styles.ts` for consistency
- Gold highlight on hover (`rgba(200, 146, 42, 0.15)`)
- Lighter gold for selected item (`rgba(200, 146, 42, 0.08)`)
- Neumorphic dropdown shadow (`neu(8)`)
- Animated chevron rotation (0.2s transition)

## Code Review — Consistency Fix

After initial implementation, a code review identified inconsistent label formats between BlogPostForm and ProjectForm. Fixed by updating BlogPostForm to use the same bilingual label format as ProjectForm:

**Before:** `locale === 'zh' ? p.titleZh : p.titleEn`
**After:** `locale === 'zh' ? \`${p.titleZh} / ${p.titleEn}\` : \`${p.titleEn} / ${p.titleZh}\``

This ensures admins can verify they're selecting the correct item regardless of their locale preference.

## Google Analytics 4 Integration

Added comprehensive GA4 tracking to the application.

### New Files

| File | Description |
|------|-------------|
| `components/GoogleAnalytics.tsx` | GA4 script loader component using `next/script` |
| `lib/analytics.ts` | Event tracking utilities for forms, phone clicks, CTAs, etc. |

### GoogleAnalytics Component

Client component that loads the Google Analytics 4 tracking script:
- Uses `next/script` with `afterInteractive` strategy for optimal loading
- Conditionally renders only when `NEXT_PUBLIC_GA_MEASUREMENT_ID` is set
- Automatically tracks pageviews via gtag config

### Analytics Utility Functions

```typescript
trackEvent(eventName, params?)         // Generic custom event
trackFormSubmission(formName, params?) // Form submission (contact, etc.)
trackPhoneClick(phoneNumber)           // Phone link clicks
trackExternalLinkClick(url, linkType?) // External links (products, social)
trackCtaClick(ctaName, location?)      // CTA button clicks
trackProjectView(slug, title?)         // Project page views
trackBlogView(slug, title?)            // Blog article views
```

### Integration Points

1. **Locale Layout** (`app/[locale]/layout.tsx`)
   - Added `<GoogleAnalytics />` component in `<head>`
   - Tracks all pageviews automatically

2. **ContactForm** (`components/ContactForm.tsx`)
   - Tracks successful form submissions via `trackFormSubmission('contact')`

3. **Footer** (`components/Footer.tsx`)
   - Tracks phone link clicks via `trackPhoneClick(company.phone)`

### Environment Variable

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_GA_MEASUREMENT_ID` | GA4 Measurement ID (e.g., `G-XXXXXXXXXX`) |

### Usage

To enable tracking:
1. Create a GA4 property in Google Analytics
2. Copy the Measurement ID (starts with `G-`)
3. Add to `.env.local`: `NEXT_PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXXXXX`

Tracking is automatically disabled if the env var is not set.
