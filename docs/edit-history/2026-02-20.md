## 1. Service Icon Migration (Lucide ŌåÆ Uploadable SVGs)

### Problem

Service icons were hardcoded as Lucide component names (e.g., `"Hammer"`, `"Bath"`) in the `iconName` DB column. Frontend components imported all 6 Lucide icons and resolved them via lookup maps. Adding a new service type required a code change.

### Solution

Added `iconUrl` column to the `services` table, created 6 SVG icon files from Lucide's open-source set, and replaced all Lucide icon rendering with `<img>` tags using a CSS filter for gold tinting.

### DB Schema Change

`lib/db/schema.ts` ŌĆö Added `iconUrl: varchar('icon_url', { length: 500 })` to the `services` table. The old `iconName` column is retained but no longer used.

### SVG Icon Files

Created `public/icons/services/` with 6 Lucide-sourced SVGs (24├Ś24 viewBox, `stroke="currentColor"`, ISC license):

| File | Lucide Source | Service |
|------|---------------|---------|
| `hammer.svg` | Hammer | kitchen |
| `bath.svg` | Bath | bathroom |
| `home.svg` | Home | whole-house |
| `arrow-down.svg` | ArrowDown | basement |
| `paintbrush.svg` | Paintbrush | cabinet |
| `building-2.svg` | Building2 | commercial |

### CSS Icon Tinting

Added `GOLD_ICON_FILTER` constant to `lib/theme.ts`:

```typescript
export const GOLD_ICON_FILTER = 'invert(56%) sepia(72%) saturate(467%) hue-rotate(6deg) brightness(90%) contrast(87%)';
```

Applied to all `<img>` tags rendering service icons to match the previous Lucide gold color (`#C8922A`).

### Query Layer

`lib/db/queries.ts` ŌĆö Changed service icon mapping:

```typescript
// Before:
icon: row.iconName ?? undefined,
// After:
icon: row.iconUrl ? getAssetUrl(row.iconUrl) : undefined,
```

### Frontend Components (3 files)

All three follow the same pattern: removed Lucide imports + icon lookup maps, replaced with `<img>` + `GOLD_ICON_FILTER`. Icon containers are conditionally rendered only when `service.icon` exists.

- `components/pages/ServicesPage.tsx` ŌĆö Removed `iconMap`, uses `<img>` with filter
- `components/pages/ServiceDetailPage.tsx` ŌĆö Removed `iconMap`, uses `<img>` with filter

## 10. Collapse Long Lists in Admin Forms

Commit: `2bd1b8f`

### Problem

Admin forms with many image pairs, service scopes, or external products became very long, making it difficult to scroll past these sections.

### Solution

Added "show first 3, then Show All" collapse pattern to three list sections:

1. **ImagePairEditor** ŌĆö Shows first 3 image pair cards by default with "Show All (N)" toggle. Hidden form inputs always render for ALL pairs to preserve form submission.
2. **ProjectForm Service Scopes** ŌĆö Same pattern. Hidden inputs for all scopes; visible cards for first 3.
3. **ProjectForm External Products** ŌĆö Same pattern. Hidden inputs for all products; visible cards for first 3.

### Implementation Details

- `COLLAPSE_THRESHOLD = 3` constant
- `showAllPairs` / `showAllScopes` / `showAllProducts` state (default `false`)
- Hidden form inputs render in a separate `.map()` loop for ALL items (always present for FormData)
- Visible cards render from `showAll ? items : items.slice(0, COLLAPSE_THRESHOLD)`
- Toggle button shows "Show All (N)" with `ChevronDown` or "Show Less" with `ChevronUp`
- Button styled with navy color to distinguish from the gold "Add" buttons

### Translation Keys Added

- `common.showAll: 'Show All ({count})'` / `'µśŠńż║Õģ©ķā© ({count})'`
- `common.showLess: 'Show Less'` / `'µöČĶĄĘ'`
