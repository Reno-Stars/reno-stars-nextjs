## 1. Service Icon Migration (Lucide → Uploadable SVGs)

### Problem

Service icons were hardcoded as Lucide component names (e.g., `"Hammer"`, `"Bath"`) in the `iconName` DB column. Frontend components imported all 6 Lucide icons and resolved them via lookup maps. Adding a new service type required a code change.

### Solution

Added `iconUrl` column to the `services` table, created 6 SVG icon files from Lucide's open-source set, and replaced all Lucide icon rendering with `<img>` tags using a CSS filter for gold tinting.

### DB Schema Change

`lib/db/schema.ts` — Added `iconUrl: varchar('icon_url', { length: 500 })` to the `services` table. The old `iconName` column is retained but no longer used.

### SVG Icon Files

Created `public/icons/services/` with 6 Lucide-sourced SVGs (24×24 viewBox, `stroke="currentColor"`, ISC license):

| File | Lucide Source | Service |
|------|---------------|---------|
| `hammer.svg` | Hammer | kitchen |
| `bath.svg` | Bath | bathroom |
| `home.svg` | Home | whole-house |
| `arrow-down.svg` | ArrowDown | basement |
| `paintbrush.svg` | Paintbrush | cabinet |
| `building-2.svg` | Building2 | commercial |

### CSS Icon Tinting

Added `GOLD_ICON_FILTER` constant to `lib/theme.ts`:

```typescript
export const GOLD_ICON_FILTER = 'invert(56%) sepia(72%) saturate(467%) hue-rotate(6deg) brightness(90%) contrast(87%)';
```

Applied to all `<img>` tags rendering service icons to match the previous Lucide gold color (`#C8922A`).

### Query Layer

`lib/db/queries.ts` — Changed service icon mapping:

```typescript
// Before:
icon: row.iconName ?? undefined,
// After:
icon: row.iconUrl ? getAssetUrl(row.iconUrl) : undefined,
```

### Frontend Components (3 files)

All three follow the same pattern: removed Lucide imports + icon lookup maps, replaced with `<img>` + `GOLD_ICON_FILTER`. Icon containers are conditionally rendered only when `service.icon` exists.

- `components/pages/ServicesPage.tsx` — Removed `iconMap`, uses `<img>` with filter
- `components/pages/ServiceDetailPage.tsx` — Removed `iconMap`, uses `<img>` with filter
- `components/home/ServicesSection.tsx` — Removed slug-based `serviceIcons` map, uses `service.icon` directly

### Admin Form

`app/admin/(dashboard)/services/ServiceForm.tsx` — Replaced the text-input icon name picker with `<ImageUrlInput>`:

```tsx
<ImageUrlInput name="iconUrl" label={t.services.iconImage} defaultValue={defaults.iconUrl ?? ''} slug={defaults.slug || undefined} imageRole="icon" disabled={!editing} />
```

### Admin List

`app/admin/(dashboard)/services/ServicesListClient.tsx` — Added custom `render` function to the title column showing a 24×24 icon thumbnail with `GOLD_PALE` background, using `getAssetUrl()` for storage URL rewriting.

### Server Actions

`app/actions/admin/services.ts` — Replaced `iconName` with `iconUrl` in both `createService` and `updateService`. Added URL validation: `if (iconUrl && !isValidUrl(iconUrl)) return { error: 'Icon URL is not a valid URL.' };`

### Admin Translations

- `messages/admin/en.ts`: `iconName: 'Icon Name'` → `iconImage: 'Icon Image'`, removed `iconNamePlaceholder`
- `messages/admin/zh.ts`: `iconName: '图标名称'` → `iconImage: '图标图片'`, removed `iconNamePlaceholder`

### Tests

`tests/unit/actions/services.test.ts` — Updated `makeFormData` default (`iconName` → `iconUrl`), added 2 new test cases for invalid icon URL validation (create + update).

## 3. Admin UX Improvements

### Detached File Picker (`ImageUrlInput`)

`components/admin/ImageUrlInput.tsx` — The upload button was a `<button>` inside `<fieldset disabled>`, which blocked all clicks in view mode. Replaced with:

1. Upload area conditionally rendered only when `!disabled` (hidden entirely in view mode)
2. Changed from `<button>` to `<div role="button">` to avoid `<fieldset disabled>` inheritance
3. Detached file picker (`document.createElement('input')`) for the actual file selection — immune to parent disabled state
4. Added keyboard support (`Enter`/`Space` triggers file picker)

### Disabled Field Styles

- `components/admin/shared-styles.ts` — Added `boxShadow: 'none'` to `readOnlyStyle` and `readOnlyTextareaStyle`
- `app/admin/admin-responsive.css` — Added global rule: `fieldset:disabled input/textarea/select { box-shadow: none !important; }`
- `ImageUrlInput` URL input: `boxShadow: disabled ? 'none' : neuIn(3)`, `cursor: 'default'` (not `'not-allowed'`)

### Disabled Prop on ImageUrlInput

Added `disabled={!editing}` to `ImageUrlInput` in 5 admin forms: BlogPostForm, CompanyForm, GalleryItemForm, ProjectForm, SiteForm.
