**Session 1** — Renovation glossary system for AI translation accuracy. AI-generated Chinese translations were producing incorrect terms for domain-specific vocabulary (e.g., "Delta" the city being transliterated as a brand name instead of translated as "三角洲"). Added a modular glossary module (`lib/ai/glossary.ts`) with categorized EN→ZH term mappings that gets injected into all 6 AI prompts. The glossary is designed to grow over time as new terms are encountered.

### Changes Made

1. **New file**: `lib/ai/glossary.ts` — Glossary data structure with categorized `{ en, zh }` term entries and `formatGlossaryForPrompt()` function that serializes terms into a prompt-injectable text block. Result is computed once at module load and cached. Deduplicates entries by lowercase `en` key.
2. **AI content optimizer**: Updated `lib/ai/content-optimizer.ts` — imported glossary, appended glossary block to all 5 prompt constants (`BLOG_CONTENT_PROMPT`, `SHORT_TEXT_PROMPT`, `PROJECT_DESCRIPTION_PROMPT`, `SITE_DESCRIPTION_PROMPT`, `ALT_TEXT_PROMPT`).
3. **AI blog generator**: Updated `lib/ai/blog-generator.ts` — imported glossary, appended glossary block to `BLOG_GENERATION_PROMPT`.

| File | Changes |

### How to Add New Terms

Add entries to the appropriate category in `lib/ai/glossary.ts`:

```typescript
const GLOSSARY: Record<string, GlossaryTerm[]> = {
  locations: [
    { en: 'Delta', zh: '三角洲' },
    // add more locations here
  ],
  'cabinet-styles': [
    { en: 'white shaker', zh: '白色shaker style柜子' },
    // add more styles here
  ],
  // add new categories as needed
};
```

All 6 AI prompts automatically pick up new terms.

**Session 7** — QA fixes: SEO, accessibility, and rating display. External QA review identified issues with duplicate carousel content, breadcrumb markup, heading semantics, and missing rating values. Fixed 4 code-level issues across 11 files.

### Changes Made

1. **Carousel SEO/a11y fix** (`TestimonialsSection.tsx`, `PartnersSection.tsx`)
   - Testimonials and partners carousels rendered all items twice via `[0, 1].map()` for seamless infinite scroll — this duplicated all links, text, and images in the DOM for crawlers and screen readers.
   - Split into two passes: first pass renders normally; second pass wrapped in `<div className="contents" aria-hidden="true" inert>` to hide from crawlers, screen readers, and keyboard navigation.
   - Visual carousel loop animation unchanged.

2. **Breadcrumb semantic fix** (`VisualBreadcrumb.tsx`)
   - Chevron separators were rendered as separate `<li>` elements inside the `<ol>`, inflating the list item count (e.g., 3-item breadcrumb had 5 `<li>` elements). Text extraction tools saw "1. 2. 3." with missing labels.
   - Moved chevrons inside their parent breadcrumb `<li>` as `aria-hidden` decorative icons. Now the `<ol>` has exactly N `<li>` elements for N breadcrumb items.
   - Removed unused `Fragment` import.

3. **Contact page heading semantics** (`ContactPage.tsx`)
   - "Request Free Consultation" was an `<h2>` with only a single `<p>` underneath — thin content under a prominent heading from a crawler's perspective.
   - Changed from `<h2>` to `<p>` with same visual styling (`text-lg font-bold`). It's a callout, not a section heading.

4. **Rating display fix** (`HeroSection.tsx`, `Footer.tsx`, `ContactPage.tsx`, `ProcessPage.tsx`, `HomePage.tsx`, `layout.tsx`, `contact/page.tsx`, `process/page.tsx`)
   - Star rating badges showed only the label "Rating" or "Google Reviews" without the actual numeric value. The `googleRating` from `getGoogleReviews()` was fetched but never passed to display components.
   - Added `googleRating?: number` prop to `HeroSection`, `Footer`, `ContactPage`, and `ProcessPage`.
   - Wired rating data: `layout.tsx` → Footer, `HomePage.tsx` → HeroSection, `contact/page.tsx` and `process/page.tsx` now fetch `getGoogleReviews()` via `Promise.all`.
   - Display: shows `"5 Rating"` when rating available, falls back to label-only when not.
   - Used `??` (nullish coalescing) for aria-label fallbacks to correctly handle `0` rating (unlike `||` which would wrongly fall back on `0`).
