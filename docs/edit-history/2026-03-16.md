# Edit History: 2026-03-16

## Summary

**Session 1** — Renovation glossary system for AI translation accuracy. AI-generated Chinese translations were producing incorrect terms for domain-specific vocabulary (e.g., "Delta" the city being transliterated as a brand name instead of translated as "三角洲"). Added a modular glossary module (`lib/ai/glossary.ts`) with categorized EN→ZH term mappings that gets injected into all 6 AI prompts. The glossary is designed to grow over time as new terms are encountered.

### Changes Made

1. **New file**: `lib/ai/glossary.ts` — Glossary data structure with categorized `{ en, zh }` term entries and `formatGlossaryForPrompt()` function that serializes terms into a prompt-injectable text block. Result is computed once at module load and cached. Deduplicates entries by lowercase `en` key.
2. **AI content optimizer**: Updated `lib/ai/content-optimizer.ts` — imported glossary, appended glossary block to all 5 prompt constants (`BLOG_CONTENT_PROMPT`, `SHORT_TEXT_PROMPT`, `PROJECT_DESCRIPTION_PROMPT`, `SITE_DESCRIPTION_PROMPT`, `ALT_TEXT_PROMPT`).
3. **AI blog generator**: Updated `lib/ai/blog-generator.ts` — imported glossary, appended glossary block to `BLOG_GENERATION_PROMPT`.

### How to Add New Terms

Add entries to the appropriate category in `lib/ai/glossary.ts`:

```typescript
const GLOSSARY: Record<string, GlossaryTerm[]> = {
  locations: [
    { en: 'Delta', zh: '三角洲' },
    // add more locations here
  ],
  'cabinet-styles': [
    { en: 'white shaker', zh: '白色shaker style柜子' },
    // add more styles here
  ],
  // add new categories as needed
};
```

All 6 AI prompts automatically pick up new terms.

**Session 2** — QA review fixes. External review identified content/UX/SEO issues across the public site. Fixed 6 actionable issues: project count pluralization, contact page duplicate business hours, process page duplicate heading, process page Chinese subtitle in English, area page missing breadcrumb, and hardcoded years of experience in meta descriptions.

### Changes Made

1. **Project count pluralization** (`ProjectsPage.tsx`, `messages/{en,zh}.json`)
   - `filter.projects` now uses ICU plural format: EN `{count, plural, one {project} other {projects}}`, ZH `{count, plural, one {个项目} other {个项目}}`
   - Component passes `{ count }` param — count displayed separately, ICU outputs word only (no `#`)
   - Fixes "Commercial 1 projects" → "Commercial 1 project"

2. **Contact page business hours** (`ContactPage.tsx`, `messages/{en,zh}.json`)
   - Quick contact bar now shows condensed hours via new `label.businessHoursShort` key ("Mon–Fri 8AM–6PM, Sat 9AM–4PM") instead of label-only "Business Hours"
   - Full details still appear in contact info cards section below

3. **Process page duplicate heading** (`messages/{en,zh}.json`)
   - `process.hero.subtitle` changed from identical copy of title to "From consultation to completion" / "从咨询到完工"
   - Fixed zh.json subtitle which was incorrectly in English

4. **Area page breadcrumbs** (`AreaPage.tsx`)
   - Added missing middle breadcrumb: `{ href: '/areas', label: t('nav.areas') }`
   - Now matches 3-item BreadcrumbSchema: Home → Areas → City

5. **Dynamic years of experience** (`queries.ts`, `messages/{en,zh}.json`, 3 page files)
   - Changed rounding from `Math.round` to `Math.ceil` (rounds up to nearest 5)
   - Replaced 8 hardcoded "27+" instances with `{years}` placeholder in meta descriptions and benefits title
   - Updated `generateMetadata` in `page.tsx` (home), `benefits/page.tsx`, and `services/[service-slug]/[city]/page.tsx` to fetch company and pass `{ years }` param
   - All three use `Promise.all` to parallelize `getTranslations` + `getCompanyFromDb`
