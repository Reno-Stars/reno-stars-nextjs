# Edit History: 2026-03-16

## Summary

**Session 1** — Renovation glossary system for AI translation accuracy. AI-generated Chinese translations were producing incorrect terms for domain-specific vocabulary (e.g., "Delta" the city being transliterated as a brand name instead of translated as "三角洲"). Added a modular glossary module (`lib/ai/glossary.ts`) with categorized EN→ZH term mappings that gets injected into all 6 AI prompts. The glossary is designed to grow over time as new terms are encountered.

### Changes Made

1. **New file**: `lib/ai/glossary.ts` — Glossary data structure with categorized `{ en, zh }` term entries and `formatGlossaryForPrompt()` function that serializes terms into a prompt-injectable text block. Result is computed once at module load and cached. Deduplicates entries by lowercase `en` key.
2. **AI content optimizer**: Updated `lib/ai/content-optimizer.ts` — imported glossary, appended glossary block to all 5 prompt constants (`BLOG_CONTENT_PROMPT`, `SHORT_TEXT_PROMPT`, `PROJECT_DESCRIPTION_PROMPT`, `SITE_DESCRIPTION_PROMPT`, `ALT_TEXT_PROMPT`).
3. **AI blog generator**: Updated `lib/ai/blog-generator.ts` — imported glossary, appended glossary block to `BLOG_GENERATION_PROMPT`.

### Initial Glossary Terms

|----------|---------|---------|

| cabinet-styles | white shaker | 白色shaker style柜子 |

### Files Modified

| File | Changes |
|------|---------|

### How to Add New Terms

Add entries to the appropriate category in `lib/ai/glossary.ts`:

```typescript
const GLOSSARY: Record<string, GlossaryTerm[]> = {
  locations: [
    { en: 'Delta', zh: '三角洲' },
    // add more locations here
  ],
  'cabinet-styles': [
    { en: 'white shaker', zh: '白色shaker style柜子' },
    // add more styles here
  ],
  // add new categories as needed
};
```

All 6 AI prompts automatically pick up new terms.

**Session 3** — Add Space Type to Sites. Sites (`projectSites` table) previously had no space type field — it only existed on projects. Added `spaceTypeEn`/`spaceTypeZh` columns to `projectSites` and propagated through the full stack: types, queries, localization, admin form, server action, and three public display components.

### Changes Made

1. **Database schema** (`lib/db/schema.ts`) — Added `space_type_en` and `space_type_zh` varchar(100) columns to `projectSites` table.

2. **TypeScript types** (`lib/types.ts`) — Added `space_type?: Localized<string>` to `Site` interface and `space_type?: string` to `LocalizedSite` interface.

3. **Query mapping** (`lib/db/queries.ts`) — Added `space_type` mapping in `mapDbSiteToSite()` using `||` pattern (consistent with `duration`). Also fixed project-side `mapDbProjectToProject()` to use `||` instead of `&&` for consistency.

4. **Localization** (`lib/data/projects.ts`) — Added `space_type: site.space_type?.[locale]` in `getLocalizedSite()`.

5. **Admin form** (`components/admin/SiteForm.tsx`) — Added space type dropdown reusing `SPACE_TYPES` from `lib/admin/constants.ts`. Includes state, sync on initialData change, and tooltip.

6. **Server action** (`app/actions/admin/sites.ts`) — Extracts `spaceType` from form data, auto-translates EN→ZH via `SPACE_TYPE_TO_ZH` lookup.

7. **Admin pages** (`sites/[id]/page.tsx`, `SiteDetailClient.tsx`) — Pass `spaceTypeEn` through to client component.

8. **Public: Projects listing** (`ProjectsPage.tsx`) — Sites now include `space_type` when converted to `DisplayProject`. Space type filter includes site space types. Filter option labels and localized lookup check both projects and sites.

9. **Public: Project modal** (`ProjectModal.tsx`) — Removed `isSite ? undefined : ...` suppression — sites now show their own space type.

10. **Public: Site detail page** (`SiteDetailPage.tsx`) — Added space type stat card with Home icon in the aggregated stats grid.

11. **Admin i18n** (`messages/admin/en.ts`, `zh.ts`) — Added `sites.spaceType`, `sites.selectSpaceType`, and `sites.tooltips.spaceType` keys (site-specific namespace, not reusing `projects.*`).
