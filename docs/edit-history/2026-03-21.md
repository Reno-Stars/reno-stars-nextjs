# 2026-03-21

## Session 1: SEO Quick Wins — Homepage H1 Keywords, Area Content Rendering, Meta Title Differentiation

Three targeted SEO improvements based on external SEO engineer audit feedback. Changed homepage H1 from brand-focused to keyword-focused, enabled rendering of already-seeded area content/highlights on service+location pages, and differentiated meta titles using service tags.

### Changes

#### 1. Homepage H1 keyword optimization
- **Files:** `messages/en.json`, `messages/zh.json`
- Changed `hero.transformYourSpace` from brand copy to target keywords:
  - EN: `"Transform Your Space"` → `"Vancouver Home Renovation Experts"`
  - ZH: `"焕新您的空间"` → `"温哥华专业家居装修"`
- Subtitle already mentions "Vancouver and the Lower Mainland" with experience stats, creating strong H1 + supporting description combination

#### 2. Area content section on ServiceLocationPage
- **File:** `components/pages/ServiceLocationPage.tsx`
- Added new "Area Content" section after service description, rendering `localizedArea.content` when present
- Content split by `\n\n` into paragraphs (same pattern as service long_description)
- Uses distinct H2 via new `areas.areaServiceContent` translation key (avoids duplicate H2 with sr-only `areas.aboutService` heading)
- Background: `SURFACE_ALT` (distinct from service description's `SURFACE`)

#### 3. Area highlights in "Why Choose" section
- **File:** `components/pages/ServiceLocationPage.tsx`
- "Why Choose [Area]" section now uses `localizedArea.highlights` when available (non-empty array)
- Falls back to hardcoded generic `benefits` list when highlights are empty or missing
- Explicit length check: `(localizedArea.highlights?.length ?? 0) > 0`

#### 4. Meta title differentiation with service tags
- **Files:** `app/[locale]/services/[service-slug]/[city]/page.tsx`, `messages/en.json`, `messages/zh.json`
- `generateMetadata` grabs first 2 service tags and joins with " & " to create a tagline
- New `titleWithTagline` template: EN `"{service} in {area} – {tagline} | Reno Stars"`, ZH `"{area}{service} – {tagline} | Reno Stars"`
- Tagline variant only used when total title length ≤ 60 chars (Google truncation limit)
- Falls back to original template when tagline is empty or title would be too long

#### 5. Translation keys added
- **Files:** `messages/en.json`, `messages/zh.json`
- `areas.areaServiceContent`: EN `"{service} Services in {area}"`, ZH `"{area}{service}服务详情"`
- `metadata.serviceLocation.titleWithTagline`: EN `"{service} in {area} – {tagline} | Reno Stars"`, ZH `"{area}{service} – {tagline} | Reno Stars"`

#### Code review fixes (2 rounds)

##### Round 1
- Changed area content H2 from `areas.aboutService` (duplicate of sr-only H2) to new `areas.areaServiceContent` key
- Changed area content background from `SURFACE` to `SURFACE_ALT` (was same as service description above)
- Made highlights check more explicit: `(?.length ?? 0) > 0` with non-null assertion

##### Round 2
- Clean — 0 actionable issues (2 cosmetic suggestions acknowledged as acceptable)
