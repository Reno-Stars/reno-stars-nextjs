# 2026-03-23 — Service Scope & SEO Fallback Fixes + SEO Audit Fixes

## Summary

Fixed three bugs causing service scopes and English SEO fields to be missing after AI analysis and batch upload. Extracted shared scope constants to eliminate code duplication. Added bilingual scope names to AI prompt for reliable Chinese-note matching.

## Bug Fixes

### Service Scope missing from AI analyze (ProjectForm)
- `components/admin/ProjectForm.tsx`: `handleAIGenerate` now applies `selectedScopes` from the AI response. Previously all AI fields except scopes were applied, leaving the form's scopes unchanged.
- Uses shared `SCOPE_EN_TO_ZH` constant for EN→ZH lookup.

### Service Scope missing from batch upload
- `lib/batch/batch-processor.ts`: `generateProjectMetadata()` now passes `ALL_SCOPES` (deduplicated across all types) when `serviceType` is null, so the AI always receives scopes to choose from.
- `lib/batch/batch-processor.ts`: `saveProjectFromUrls()` now resolves AI-selected scope names via `SCOPE_EN_TO_ZH` (all types) instead of filtering against a single service type's list. Falls back to type-specific scopes if all AI names were invalid.

### English SEO fields missing when Chinese ones are present
- `lib/ai/content-optimizer.ts`: Added `fillMissingSeoFields()` helper that auto-derives empty SEO fields (metaTitle, metaDescription, focusKeyword, seoKeywords) from titleEn/descriptionEn. Applied to both project validation (`validateProjectDescription`) and site description generation (`optimizeSiteDescription`).
- Added `truncateAtWord()` helper that breaks at sentence or word boundaries instead of cutting mid-word when deriving metaDescription.

## Refactoring

### Shared scope constants (`lib/admin/constants.ts`)
- Extracted `ALL_SCOPES` (deduplicated `{ en, zh }[]` across all service types) — previously duplicated as a local IIFE in `optimize-content.ts`.
- Extracted `SCOPE_EN_TO_ZH` (`ReadonlyMap<string, string>`) — previously built inline as `masterLookup` in both `ProjectForm.tsx` and `batch-processor.ts`.
- `app/actions/admin/optimize-content.ts`: Removed local `ALL_SCOPES` duplicate, now imports from shared constants.

### AI scope prompt shows only English names — Chinese notes can't match
- `lib/ai/content-optimizer.ts`: `AVAILABLE_SCOPES` in the AI prompt now includes both EN and ZH names (e.g., `Flooring (地面工程)` instead of just `Flooring`), so the AI can reliably match Chinese notes like `地板工程` to the correct English scope name.

## Minor

- `lib/ai/glossary.ts`: Added "Surrey" / "素里" to the location glossary.

## Session 2 — SEO Audit & Fixes

### Summary

Ran a comprehensive SEO audit against the live site. Fixed canonical URL domain mismatch, added RSS feed for blog content, added dynamic OG image generation endpoint, and updated 9 hub page metadata to use branded OG images.

### Canonical URL Domain Mismatch (Critical Fix)

- **Problem:** All HTML canonical tags, sitemap URLs, OG URLs, and hreflang alternates used `https://reno-stars.com` (non-www), but the serving domain is `https://www.reno-stars.com` (307 redirect from non-www to www). Conflicting signals to Google.
- **Fix:** Updated Vercel production env var `NEXT_PUBLIC_BASE_URL` to `https://www.reno-stars.com` via CLI. The code fallback in `lib/utils.ts` was also updated (commit `bacfdc9`).
- **Note:** Non-www → www redirect is still 307 (Vercel dashboard domain config). Should be changed to 308/301 in Vercel Dashboard → Settings → Domains.

## Session 3 — Scope Matching Bug Fix

### Summary

Fixed a bug where batch upload only matched 1 of 6 Chinese scope terms from project notes. The AI was failing to fuzzy-match Chinese scope names (e.g., "地板工程" vs "地面工程"), and the validation layer only accepted exact English name matches, silently dropping anything else.

### Root Cause

Three-layer failure:
1. **AI prompt** lacked fuzzy-matching instructions for Chinese notes
2. **Validation** (`validateProjectDescription`) only did exact English name matching — Chinese names were silently dropped
3. **Batch processor** (`saveProjectFromUrls`) had the same exact-EN-match filter
4. **Missing scope** — "Countertop Replacement" / "台面更换" didn't exist in `service-scopes.json`

### Fixes

#### ZH→EN reverse lookup (`lib/admin/constants.ts`)
- Added `SCOPE_ZH_TO_EN` (`ReadonlyMap<string, string>`) — reverse of `SCOPE_EN_TO_ZH`

#### AI prompt improvement (`lib/ai/content-optimizer.ts`)
- Added explicit fuzzy-matching instructions: when notes are in Chinese, match partial/synonym terms against parenthetical Chinese names in `AVAILABLE_SCOPES` (e.g., "地板工程" → "Flooring (地面工程)", "全屋油漆" → "Painting (油漆)")

#### Validation auto-translate (`lib/ai/content-optimizer.ts`)
- `validateProjectDescription` now builds a local `zhToEn` map from `availableScopes`
- If a scope name isn't in the valid English set, checks the ZH→EN map and auto-translates
- Logs translated scope names in corrections array for debugging
- Added deduplication to prevent duplicates when AI returns both EN and ZH for the same scope

#### Batch processor ZH support (`lib/batch/batch-processor.ts`)
- Scope resolution now checks both `SCOPE_EN_TO_ZH` and `SCOPE_ZH_TO_EN`
- Uses canonical ZH value from `SCOPE_EN_TO_ZH` (not raw AI input) for database storage

#### New scope (`lib/data/service-scopes.json`)
- Added `"Countertop Replacement"` / `"台面更换"` to kitchen service type
