# 2026-03-23 ŌĆö Service Scope & SEO Fallback Fixes

## Summary

Fixed three bugs causing service scopes and English SEO fields to be missing after AI analysis and batch upload. Extracted shared scope constants to eliminate code duplication. Added bilingual scope names to AI prompt for reliable Chinese-note matching.

## Bug Fixes

### Service Scope missing from AI analyze (ProjectForm)
- `components/admin/ProjectForm.tsx`: `handleAIGenerate` now applies `selectedScopes` from the AI response. Previously all AI fields except scopes were applied, leaving the form's scopes unchanged.
- Uses shared `SCOPE_EN_TO_ZH` constant for ENŌåÆZH lookup.

### Service Scope missing from batch upload
- `lib/batch/batch-processor.ts`: `generateProjectMetadata()` now passes `ALL_SCOPES` (deduplicated across all types) when `serviceType` is null, so the AI always receives scopes to choose from.
- `lib/batch/batch-processor.ts`: `saveProjectFromUrls()` now resolves AI-selected scope names via `SCOPE_EN_TO_ZH` (all types) instead of filtering against a single service type's list. Falls back to type-specific scopes if all AI names were invalid.

### English SEO fields missing when Chinese ones are present
- `lib/ai/content-optimizer.ts`: Added `fillMissingSeoFields()` helper that auto-derives empty SEO fields (metaTitle, metaDescription, focusKeyword, seoKeywords) from titleEn/descriptionEn. Applied to both project validation (`validateProjectDescription`) and site description generation (`optimizeSiteDescription`).
- Added `truncateAtWord()` helper that breaks at sentence or word boundaries instead of cutting mid-word when deriving metaDescription.

## Refactoring

### Shared scope constants (`lib/admin/constants.ts`)
- Extracted `ALL_SCOPES` (deduplicated `{ en, zh }[]` across all service types) ŌĆö previously duplicated as a local IIFE in `optimize-content.ts`.
- Extracted `SCOPE_EN_TO_ZH` (`ReadonlyMap<string, string>`) ŌĆö previously built inline as `masterLookup` in both `ProjectForm.tsx` and `batch-processor.ts`.
- `app/actions/admin/optimize-content.ts`: Removed local `ALL_SCOPES` duplicate, now imports from shared constants.

### AI scope prompt shows only English names ŌĆö Chinese notes can't match
- `lib/ai/content-optimizer.ts`: `AVAILABLE_SCOPES` in the AI prompt now includes both EN and ZH names (e.g., `Flooring (Õ£░ķØóÕĘźń©ŗ)` instead of just `Flooring`), so the AI can reliably match Chinese notes like `Õ£░µØ┐ÕĘźń©ŗ` to the correct English scope name.
