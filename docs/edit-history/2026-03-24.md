# 2026-03-24 — Hero Slideshow Bug Fixes (Type Error + Stale Closure + A11y)

## Summary

Fixed three issues in the hero slideshow feature: a TypeScript type mismatch causing build failure, a stale closure race condition in the auto-advance interval, and screen reader instability from a rotating `aria-labelledby` heading.

## Bug Fixes

### TypeScript type error in HomePage hero translations
- `components/pages/HomePage.tsx`: Added missing `realEstateTitle` and `realEstateDesc` to the `translations.hero` type in `HomePageProps`. The type was out of sync with both `HeroSectionProps` (which required them) and `app/[locale]/page.tsx` (which already passed them). Type-only gap introduced when the slideshow feature was added.

### Stale closure race in hero auto-advance interval
- `components/home/HeroSection.tsx`: Added `currentRef` (useRef) that is written synchronously in `goTo()` and read by the interval callback. The interval no longer depends on `current` state, so it stays stable across slide changes — no teardown/re-creation per slide, and no double-fire during the 300ms fade transition. The `setCurrent` state call still drives renders; ref and state stay in sync because every code path updates both.

### Screen reader heading instability from rotating h1
- `components/home/HeroSection.tsx`: Replaced `aria-labelledby="hero-title"` on the section with a stable `aria-label={t.transformYourSpace}` (the primary business heading). Added `aria-live="polite"` on the slide content wrapper so screen readers announce slide changes without interruption. Removed orphaned `id="hero-title"` from the h1.

## Files Changed
