# 2026-03-18

## Session 1: Batch Upload — Unsupported Format Tracking, ZIP Context Hints, Better Fallback Metadata

Three features added to the batch upload pipeline, followed by code review fixes.

### New Features (pre-review changes)

#### 1. Unsupported image format tracking
- **Files:** `lib/batch/types.ts`, `lib/batch/zip-parser.ts`, `lib/batch/batch-processor.ts`
- Added `UNSUPPORTED_IMAGE_EXTENSIONS` set (heic, heif, tiff, tif, bmp, raw) and `isUnsupportedImageFile()` helper in zip-parser
- Both `parseZip()` and `parseZipStandalone()` now collect skipped filenames into a `skippedFiles` array
- Added `skippedFiles: string[]` field to both `ParsedZipStructure` and `ParsedStandaloneStructure` interfaces
- `pushSkippedFilesWarning()` surfaces skipped files as warnings in the job error list

#### 2. ZIP filename as PO number hint (standalone mode)
- **Files:** `lib/batch/batch-processor.ts`
- Standalone mode extracts `zipBaseName` from the uploaded filename (e.g., "2828-van.zip" → "2828-van")
- For single-project ZIPs, the base name is passed to AI metadata generation as a `Reference/PO number` context hint
- Also passed through `saveProject()` as `zipBaseName` for fallback PO number usage

#### 3. Better fallback metadata via `ensureActionSuffix()`
- **Files:** `lib/batch/batch-processor.ts`
- New `ensureActionSuffix(label, zh?)` function ensures labels include action words (Renovation/装修/翻新/改造)
- Refactored `buildFallbackSeo()` to accept pre-computed `fullEn`/`fullZh` labels instead of raw name + type
- `fallbackSiteData()` and `fallbackProjectData()` now use `ensureActionSuffix()` for consistent labels
- Removed generic `FALLBACK_ALT` constant — site image pair alt text now uses contextual titles (e.g., "Kitchen Renovation - Before 1")
- Fallback site descriptions no longer include folder name (often internal codes)

### Code Review Fixes

#### 4. Removed unused `_folderName` parameter from `fallbackProjectData()`
- **Files:** `lib/batch/batch-processor.ts`
- Parameter was made unused by the refactor but kept with underscore prefix — removed entirely along with the argument at the call site

#### 5. Removed redundant `singleProjectZipBase` alias
- **Files:** `lib/batch/batch-processor.ts`
- Was just an alias of `aiZipHint` with no transformation — now uses `aiZipHint` directly

#### 6. Truncated long skipped files warnings
- **Files:** `lib/batch/batch-processor.ts`
- `pushSkippedFilesWarning()` now lists max 10 filenames, with "and N more" suffix for overflow
- Prevents unbounded warning messages when ZIPs contain many HEIC/TIFF files

#### 7. Added word boundaries to `ensureActionSuffix` EN regex
- **Files:** `lib/batch/batch-processor.ts`
- Changed `/renovation|refacing|remodel/i` to `/\b(renovation|refacing|remodel)\b/i` for more precise matching

#### 8. Added unit tests for new utility functions
- **Files:** `tests/unit/batch/batch-processor.test.ts`
- 6 tests for `ensureActionSuffix` (EN append, EN skip, case-insensitive, word boundary, ZH append, ZH skip)
- 5 tests for `pushSkippedFilesWarning` (empty, basic, path stripping, truncation at 10, no "more" at exactly 10)
- Both functions exported for testability

## Session 2: AI Output Review — Hybrid Validation for Project Descriptions

Added a two-layer validation system for AI-generated project descriptions to catch invalid scopes, bad service types, and slug-title mismatches.

### New Features

#### 1. Programmatic validation (`validateProjectDescription`)
- **Files:** `lib/ai/content-optimizer.ts`
- Pure function that runs immediately after Zod parsing in `optimizeProjectDescription()`
- **Scope enforcement:** Filters `selectedScopes` to only entries existing in the provided scope list; falls back to all scopes if every AI-selected scope was invalid
- **ServiceType enforcement:** Sets `serviceType` to `null` if not in the allowed list
- **Slug-title consistency:** Regenerates slug from title via `formatSlug()` when slug and title share fewer than `MIN_SLUG_TITLE_OVERLAP` (2) significant words (stop words excluded)
- Returns `{ corrected, corrections }` — corrections array used for logging
- Deep-copies `selectedScopes` array to avoid mutating input

#### 2. Lightweight AI review (`reviewProjectDescription`)
- **Files:** `lib/ai/content-optimizer.ts`
- Second, cheaper AI call using `gpt-4o-mini` with `temperature: 0.1` and 512 max tokens
- Sends only the raw notes + key generated fields (title, serviceType, slug, scopes) to a focused reviewer prompt
- Returns only fields that need correction as a partial object; returns `null` if everything is correct
- Non-fatal — wrapped in try/catch, failures just log a warning and skip the review
- `ReviewCorrectionsSchema` uses strict Zod (no `.passthrough()`) to drop unknown fields

#### 3. Integration into `optimizeProjectDescription()`
- **Files:** `lib/ai/content-optimizer.ts`
- After Zod parse: (1) programmatic validation always runs, (2) AI review runs and applies corrections on top
- Review corrections are re-validated (serviceType checked against allowed list, slug sanitized through `formatSlug()`, scopes re-validated)
- Return type extended to `ProjectDescription & { corrections?: string[] }`
- All corrections logged via `console.log('[AI Validation]', ...)`

#### 4. Batch processor logging
- **Files:** `lib/batch/batch-processor.ts`
- `generateProjectMetadata()` logs corrections from the validation/review step
- Existing scope filter at save time (lines 1098-1103) remains as a safety net

#### 5. `maxTokensReview` config
- **Files:** `lib/ai/openai.ts`
- Added `maxTokensReview: 512` to `AI_CONFIG`

### Code Review Fixes

#### 6. ServiceType re-validation after AI review
- AI review could return a serviceType not in the allowed list — now re-validated after applying review corrections

#### 7. Slug sanitization after AI review
- Reviewer-provided slug is run through `formatSlug()` to ensure valid URL format

#### 8. Removed `.passthrough()` from ReviewCorrectionsSchema

#### 12. Single-pass scope partition
