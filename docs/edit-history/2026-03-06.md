## Product Image Upload Support (Batch Upload)

### New Feature: `product-N.jpg` in ZIP folders

Users can now include `product-N.jpg` files alongside `products.txt` in any ZIP folder. The 1-based index matches the Nth product entry in `products.txt`. Sparse indices are supported (e.g., `product-1.jpg` and `product-3.jpg` â€” product 2 gets no image).

**Files modified:**

| File | Changes |
|------|---------|
| `lib/batch/types.ts` | Added `productImages: Map<number, ExtractedImage>` to both `ParsedProject` and `ParsedSite` |
| `lib/batch/zip-parser.ts` | `pairImages()` now detects `product-N` pattern via regex, returns `productImages` Map. Flat/single-folder layouts give site `new Map()` and `null` productsText to prevent duplicate uploads |
| `lib/batch/batch-processor.ts` | Product images added to S3 upload batch. Extracted shared `insertExternalProducts()` helper for both site-level and project-level insertion with product image matching. Warning logged when product images exist but no `products.txt` |

### Shared reference bug fix

In flat and single-folder ZIP layouts, the synthetic site wrapper shared the same `productImages` Map and `productsText` with its child project, causing duplicate S3 uploads and duplicate DB rows. Fixed by giving the site `new Map()` and `null` productsText.

### `product-0` guard

Added `productIdx >= 1` check to prevent `product-0.jpg` from being accepted (would never match any 1-based product entry).
