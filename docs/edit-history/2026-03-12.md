# 2026-03-12 — Batch Upload R2 Compatibility + Performance + Stale Job Detection

## Summary

Made batch upload compatible with Cloudflare R2 by disabling AWS SDK v3 automatic CRC32 checksums and switching from XHR to fetch() for presigned URL uploads. Parallelized S3 uploads (15 concurrent, was 3) and AI metadata generation (10 concurrent, was sequential). Removed alt text generation entirely to fit Vercel Hobby 60s timeout. Added stale job detection with bilingual timeout error messages. Improved client error handling with validated API response types.

## Changes

### S3 Client — CRC32 Checksum Fix

- **`lib/admin/s3.ts`** — Added `requestChecksumCalculation: 'WHEN_REQUIRED'` to S3Client config. Disables AWS SDK v3 automatic CRC32 checksum headers on presigned URLs, which R2 rejects as unsigned headers.

### Batch Upload Client — XHR to fetch()

- **`app/admin/(dashboard)/batch-upload/BatchUploadClient.tsx`** — Replaced XHR-based ZIP upload with `fetch()` PUT for R2 compatibility (XHR added headers that conflicted with presigned URL signatures). Polling now starts immediately after job creation (before S3 upload completes) so the UI always transitions to results phase. On S3 upload failure, client returns to upload phase; if only the process trigger POST fails, polling continues (processing may already be running via `after()`). Added `resolveErrorLabel()` helper that maps `__TIMEOUT_*__` marker strings from the server into bilingual display text (`timeoutPartial`, `timeoutFailed`, `timeoutStep` translation keys). Validated all API response types with runtime checks (removed `as` type casts).

### Stale Job Detection

- **`app/admin/(dashboard)/batch-upload/api/[jobId]/route.ts`** — Added stale job detection in the GET polling endpoint. `STALE_PROCESSING_MS` (2 minutes): if a job has been in `processing` status longer than this, it's marked `partial` (if any creations exist) or `failed`. `STALE_PENDING_MS` (30 minutes): catches abandoned `pending` jobs. Stale jobs get `__TIMEOUT_*__` error markers that the client resolves into bilingual messages.

### Parallelized Processing

- **`lib/batch/batch-processor.ts`** — `UPLOAD_BATCH_SIZE` increased from 3 to 15 for concurrent S3 image uploads. Added `AI_METADATA_BATCH_SIZE = 10` for parallel AI metadata generation (previously sequential). Both sites mode and standalone mode use batched AI calls. Removed alt text generation entirely (`generateAltText`, `generateAltTextsInBatches`, `altTextMap` code all removed) — image pairs now use `FALLBACK_ALT` directly. Cleaned up dead `altTextMap` parameter from `saveProject()` function signature.

### maxDuration + API Route Updates

- **`app/admin/(dashboard)/batch-upload/api/[jobId]/process/route.ts`** — `maxDuration` changed from 300 to 60 (Vercel Hobby tier limit).
- **`app/admin/(dashboard)/batch-upload/api/route.ts`** — Added comment clarifying not to include `ContentLength` in presigned URL params for batch uploads (browsers set it automatically on fetch PUT). `maxDuration` set to 60.

### Translations

- **`messages/admin/en.ts`** — Added batch upload timeout i18n keys: `timeoutPartial`, `timeoutFailed`, `timeoutStep`.
- **`messages/admin/zh.ts`** — Added corresponding Chinese translations.

### Documentation

- **`CLAUDE.md`** — Updated batch upload section: fetch() instead of XHR, early polling, stale detection, increased batch sizes, no alt text generation, resolveErrorLabel helper. Updated S3 client description with checksum config.
- **`docs/architecture.md`** — Updated batch processor pipeline (batch sizes, removed alt text references), S3 client config, client UI description, AI config (removed alt text token config), AI functions table (removed generateAltText).
