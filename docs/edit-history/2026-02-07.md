## Blog Post → Project Reference Feature

Blog posts can now link to a related project. When a blog references a project, the blog page displays a project card and the products used in that project.

### Database Schema (`lib/db/schema.ts`)

Added `projectId` foreign key to `blogPosts` table:

```typescript
projectId: uuid('project_id').references(() => projects.id, { onDelete: 'set null' }),
```

### Types (`lib/types.ts`)

Added `BlogRelatedProject` interface and `related_project` field to `BlogPost`:

```typescript
interface BlogRelatedProject {
  slug: string;
  title: Localized<string>;
  hero_image?: string;
  external_products?: { url: string; image_url?: string; label: Localized<string> }[];
}

interface BlogPost {
  // ...existing fields
  related_project?: BlogRelatedProject;
}
```

### Query Updates (`lib/db/queries.ts`)

- `getBlogPostBySlugFromDb()` now joins with projects and project_external_products when `projectId` is set
- Returns `related_project` with slug, title, hero_image, and localized external products

### Admin Updates

- `BlogPostForm.tsx` — Added project selector dropdown
- `app/admin/(dashboard)/blog/[id]/page.tsx` — Fetches projects list for dropdown
- `app/admin/(dashboard)/blog/new/page.tsx` — Fetches projects list for dropdown
- `app/actions/admin/blog.ts` — Handles `projectId` field in form data

### Blog Page Display (`components/pages/BlogPostPage.tsx`)

Added "Related Project" section when `related_project` exists:
- Project card with hero image and link
- External products list with ProductLink component (same UI as project detail page)

## Whole House Project Modal Enhancements

Enhanced the project modal for "whole house" site projects with additional aggregated data.

### New Data in Modal

For sites displayed as projects, the modal now shows:
- **Included Areas** — List of child project categories (e.g., Kitchen, Bathroom)
- **Total Budget** — Aggregated from all child projects
- **Total Duration** — Aggregated timeline
- **Service Scopes** — Merged unique scopes from all children
- **Products Used** — All external products from child projects (deduplicated)

### Type Updates (`lib/types.ts`)

Added `DisplayProject` interface extending `LocalizedProject`:

```typescript
interface DisplayProject extends LocalizedProject {
  isSiteProject?: boolean;
  projectCount?: number;
  childAreas?: string[];
  totalBudget?: string;
  totalDuration?: string;
  allServiceScopes?: string[];
  allExternalProducts?: { url: string; image_url?: string; label: string }[];
}
```

Added `allExternalProducts` to `SiteAggregated` and `LocalizedSiteAggregated`:

```typescript
allExternalProducts: { url: string; image_url?: string; label: Localized<string> }[];
```

### Helper Functions (`lib/db/helpers.ts`)

New function for product aggregation:

```typescript
collectAllExternalProducts(projects: Project[]): SiteWithProjects['aggregated']['allExternalProducts']
```

Deduplicates products by URL across all child projects.

### Query Updates (`lib/db/queries.ts`)

- `getSitesAsProjectsFromDb()` now returns full `SiteWithProjects[]` with aggregated data
- Includes `allExternalProducts` in aggregation

### UI Updates

- `ProjectModal.tsx` — Shows aggregated data for site projects, uses ProductLink component
- `ProjectsPage.tsx` — Passes site-specific aggregated data when building DisplayProject objects
- `SiteDetailPage.tsx` — Shows external products section with ProductLink

### File: `components/ProductLink.tsx`

```typescript

}
```

## CSP Update (`proxy.ts`)

Updated Content-Security-Policy to allow external HTTPS images:

```typescript
`img-src 'self' data: blob: https: ${ASSET_ORIGIN}`
```

This allows product thumbnails from external domains (e.g., Lowe's, Home Depot).
