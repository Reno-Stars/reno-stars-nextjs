## 9. HowTo Schema on Process Page

Created `components/structured-data/HowToSchema.tsx` — renders JSON-LD `HowTo` schema with 5 renovation workflow steps:

```typescript
const schema = {
  '@context': 'https://schema.org',
  '@type': 'HowTo',
  name,
  description,
  step: steps.map((step, index) => ({
    '@type': 'HowToStep',
    position: index + 1,
    name: step.name,
    text: step.text,
  })),
  totalTime: 'P60D',
  tool: [
    { '@type': 'HowToTool', name: 'Floor plans' },
    { '@type': 'HowToTool', name: 'Budget estimate' },
  ],
  isPartOf: { '@type': 'WebSite', url: baseUrl, name: SITE_NAME },
};
```

Exported from `components/structured-data/index.ts` and rendered in `app/[locale]/process/page.tsx` using FAQ data for step names and descriptions.

## 14. Project Meta Description Prefers Excerpt

Changed the meta description fallback chain in `app/[locale]/projects/[slug]/page.tsx`:

**Before:**
```typescript
const metaDescription = project.meta_description?.[locale as Locale]
  || truncateMetaDescription(localizedProject.description);
```

**After:**
```typescript
const metaDescription = project.meta_description?.[locale as Locale]
  || truncateMetaDescription(project.excerpt?.[locale as Locale] || localizedProject.description);
```

Excerpts are typically more concise and SERP-optimized than full descriptions, making them a better fallback before the generic description.

## 15. Fixed 404 Heading Hierarchy

`app/[locale]/not-found.tsx` had a separate `<h1>` for "404" and `<h2>` for "Page Not Found", creating a broken heading hierarchy when the page had no content sections to justify an H2.

**Before:**
```tsx
<h1>404</h1>
<h2>{t('notFound')}</h2>
```

**After:**
```tsx
<h1>
  <span className="block text-6xl font-bold">404</span>
  <span className="block text-2xl font-semibold">{t('notFound')}</span>
</h1>
```

Visual appearance is unchanged; semantic structure is now correct with a single H1.
