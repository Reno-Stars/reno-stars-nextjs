## CGL Insurance / WCB Coverage Separation

### Background

Previously, "$5M CGL Insurance" and "Active WCB Coverage" were conflated into a single "third-party liability insurance" mention across the site. These are two distinct protections: CGL (Commercial General Liability) protects the homeowner's property during renovation, while WCB (WorkSafeBC) protects workers on the job site. They are now displayed as separate, independent items everywhere on the site.

### Approach

- CGL and WCB are **translation-driven** — no runtime boolean flag needed. The `Company` type does not carry a `wcbCoverage` field; display strings come from `stats.liabilityCoverage` and `stats.wcbCoverage` translation keys.
- Visual distinction: **Shield icon** (lucide-react) for CGL insurance, **WorkSafe BC logo** (`/worksafe-bc-logo.jpg`) for WCB coverage, **HardHat icon** (lucide-react) for WCB on About page stat cards.
- Empty stat values (WCB has no numeric value) handled with conditional rendering: `{value && <span>...</span>}`.

## Category Swipe Indicators (Projects Page — Mobile UX)

### Background

The horizontal category card scroll on the Projects page (`/projects`) had no visual cue on mobile that users could swipe to see more categories. Easy to miss that the row was scrollable.

### Approach

Three visual indicators added, all reactive to scroll position:

1. **Right-edge gradient fade** — shows when more content is available to the right. Fades out at end of scroll.
2. **Left-edge gradient fade** — shows when user has scrolled away from the start.
3. **"Swipe to see more →" text hint** — mobile-only (`sm:hidden`), pulses twice then fades out via a single CSS `@keyframes swipe-hint` animation (3.2s total). Rendered unconditionally with `aria-hidden="true"` to avoid remount/re-animation on scroll state changes.

### Implementation details

- `updateScrollIndicators()` callback tracks `canScrollLeft` / `canScrollRight` state from scroll position. Uses functional setState updater to skip re-renders when values haven't changed.
- `useEffect` attaches passive scroll listener + resize listener with cleanup.
- `SCROLL_THRESHOLD` constant (8px) used for edge detection buffer.
- `prefers-reduced-motion` override: disables animation, shows static hint at `opacity: 0.7`.
- Gradient overlays use `pointer-events-none` + `z-10` positioned over the scroll container, colored to match `SURFACE_ALT` background.

### Two rounds of code review — issues fixed

| # | Issue | Fix |
|---|-------|-----|
| 1 | `canScrollRight` initialized `true` causing flash before measurement | Changed to `false`; effect sets correct value on mount |
| 2 | `setState` on every scroll tick even when value unchanged | Functional updater: `(prev) => prev !== val ? val : prev` |
| 3 | Infinite `animate-pulse` distracting | Custom single-run animation (pulse 2x, fade out) |
| 4 | Two-animation timing gap (0.4s opacity snap between pulse end and fadeOut start) | Consolidated into single `@keyframes` sequence |
| 5 | Missing `prefers-reduced-motion` | Added override in `globals.css` |
| 6 | Conditional mount replayed animation on scroll toggle | Render unconditionally with `aria-hidden` |
| 7 | Magic number `8` used twice | Extracted to `SCROLL_THRESHOLD` constant |

## Before & After Page Pagination

### Background

The Before & After gallery page (`/before-after/`) displayed all image pairs at once with no pagination, causing performance issues as the number of projects grew.

### Implementation

Added client-side pagination to `BeforeAfterGalleryPage`, following the same pattern as `ProjectsPage`:

- `ITEMS_PER_PAGE = 12` (4 rows × 3 columns on desktop)
- `currentPage` state resets to 1 on filter change
- `paginatedItems` slices filtered items for current page
- Smart page number generation with ellipsis for large page counts (≤7 pages show all, >7 shows first/last with ±1 around current)
- Neumorphic-styled pagination nav: Previous/Next buttons + numbered page buttons (GOLD active, CARD inactive, `neu()` shadows)
- `galleryRef` with `scrollIntoView({ behavior: 'smooth' })` on page change
- `scroll-mt-16` on gallery section for sticky filter bar clearance
- Eager loading (`loading="eager"`) applied to first 6 items per page (uses local `idx`, not global index)
