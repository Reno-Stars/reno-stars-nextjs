# 2026-03-29 — Navbar Projects Dropdown with Service Filtering + Service Visibility Flags

## Summary

Added a dropdown menu to the "Projects" navbar item. On hover (desktop) or inline (mobile), the dropdown lists all service categories (Kitchen, Bathroom, etc.) as links to `/projects?service=<slug>`. Clicking a service navigates to the projects page with the category filter pre-applied via `?service=` search parameter. Multiple rounds of code review with all issues fixed.

Also fixed a data issue: the kitchen service slug was corrupted to `kitchen-e2e-test` (in both local and production databases) by a previous e2e test session, causing 16 kitchen projects to disappear from the category filter.

Added two independent boolean flags to the `services` table — `showOnServicesPage` and `isProjectType` — to control where each service appears across the site. Both default to `true` for backward compatibility. Admin ServiceForm includes bilingual checkbox UI. Two rounds of code review with all issues fixed.

## Navbar Dropdown

### Navbar component — dropdown for Projects
- `components/Navbar.tsx`: Added `services` prop (optional `Service[]`). The "Projects" nav item now renders a hover-activated dropdown on desktop with service categories. Mobile menu shows services as indented sub-items under "Projects".
- Desktop: `onMouseEnter`/`onMouseLeave` with 150ms close delay (`DROPDOWN_CLOSE_DELAY_MS` constant). `onKeyDown` handler for ArrowDown/ArrowUp/Escape keyboard navigation. ARIA: `role="menu"` on panel, `role="menuitem"` + `tabIndex={-1}` on items, `aria-haspopup="menu"` + `aria-expanded` on trigger.
- Mobile: Semantic `<ul role="list">` with `<li>` wrapping each service link.
- Service links use `<a>` with `router.push()` from next-intl for proper query param handling (next-intl's `Link` doesn't reliably handle query strings in href). `href` fallback for no-JS/accessibility.
- `dropdownItemsRef` array resized on each render to match service count (prevents stale refs).
- Cleanup: timeout cleared on unmount, outside-click listener only attached when open.
- Added `NavLink` interface with typed `isDropdown?: boolean`.
- Shared `isLinkActive()` callback used by both desktop and mobile (unified `pathname.startsWith()` logic for dropdown items).

### Layout — passes services to Navbar
- `app/[locale]/layout.tsx`: Added `services={services}` prop to `<Navbar>`. Services were already fetched via `getServicesFromDb()` in `Promise.all`.

## Projects Page — Search Param Filtering

### Server component — reads and validates `?service=` param
- `app/[locale]/projects/page.tsx`: Added `searchParams` to `PageProps`. Reads `resolvedSearchParams.service`, validates against known categories via whitelist check (server-side), passes validated `initialService` to `ProjectsPage`. Invalid values resolve to `undefined`.

### Client component — applies initial filter
- `components/pages/ProjectsPage.tsx`: Added `initialService` prop. `useState` initializes `activeCategory` from `initialService ?? 'All'`. `useEffect` with `isFirstRender` ref guard syncs `activeCategory` when `initialService` prop changes (handles client-side navigations where component is reused).

## Database Fix

- Fixed `kitchen` service slug corrupted to `kitchen-e2e-test` in both local Docker and production Neon databases (caused by e2e test from 2026-03-27 session not restoring original slug).
- Created missing `service_benefits` and `service_tags` tables in local Docker Postgres via `db:push` with explicit `DATABASE_URL` override (drizzle-kit reads `.env` not `.env.local`).

## Service Visibility Flags

### Schema & types
- `lib/db/schema.ts`: Added `show_on_services_page` and `is_project_type` boolean columns (default `true`, not null)
- `lib/types.ts`: Added optional `showOnServicesPage` and `isProjectType` to `Service` interface
- `drizzle/0002_chilly_blindfold.sql`: Auto-generated migration for the two new columns

### Query layer
- `lib/db/queries.ts`: `getServicesFromDb()` maps both new fields. `getServiceTypeMap()` skips services with `isProjectType === false`, cascading to `getCategoriesLocalized()` and `getCategorySlugs()`

### Public page filtering
- `app/[locale]/layout.tsx`: Navbar receives services filtered by `isProjectType !== false`; Footer receives services filtered by `showOnServicesPage !== false`
- `app/[locale]/services/page.tsx`: Filters by `showOnServicesPage !== false` before passing to component
- `app/[locale]/services/[service-slug]/page.tsx`: Skips in `generateStaticParams`, returns `notFound()` in render/metadata when `showOnServicesPage === false`
- `app/[locale]/services/[service-slug]/[city]/page.tsx`: Same three-point guard as service detail page
- `app/[locale]/page.tsx`: Homepage services section filtered by `showOnServicesPage !== false`
- `app/sitemap.ts`: Service URLs use `showOnServicesPage` filter instead of project-type slugs

### Admin
- `app/actions/admin/services.ts`: `createService()` and `updateService()` parse both checkbox values from FormData (`=== 'on'`)
- `app/admin/(dashboard)/services/ServiceForm.tsx`: Added "Visibility" `FormField` with two labeled checkboxes using translation keys
- `app/admin/(dashboard)/services/[id]/page.tsx`: Passes both flags to ServiceForm initial data
- `messages/admin/en.ts` + `zh.ts`: Added `visibility`, `showOnServicesPageLabel`, `isProjectTypeLabel` translation keys

### Seed
- `lib/db/seed.ts`: All 6 seed entries explicitly set `showOnServicesPage: true, isProjectType: true`
