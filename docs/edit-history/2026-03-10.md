# 2026-03-10 — Presigned S3 URL Upload for All Images

## Summary

Migrated all admin image uploads from server-buffered upload (5 MB limit via server action) to presigned S3 URL upload (50 MB limit, browser-to-S3 direct). This bypasses Vercel's proxy body size limit, matching the pattern already used by batch upload.

## Changes

### New Files

- **`app/api/admin/upload/route.ts`** — API route that generates presigned S3 PUT URLs for image uploads. Validates auth, file metadata (size, MIME type), generates S3 key, includes `ContentLength` condition for server-side size enforcement. 10-minute presigned URL expiry.
- **`lib/admin/upload-constants.ts`** — Shared constants (`MAX_IMAGE_SIZE` = 50 MB, `MAX_IMAGE_SIZE_LABEL`, `ALLOWED_IMAGE_TYPES`) used by both the API route and client helper.
- **`lib/admin/upload-client.ts`** — Client-side `uploadImageDirect()` function. Two-step flow: request presigned URL from API, then PUT file directly to S3. Includes early client-side validation (size + MIME type) to avoid unnecessary round-trips.

### Deleted Files

- **`app/actions/admin/upload.ts`** — Old server action that buffered entire files through the server with a 5 MB limit. No longer imported anywhere.

### Modified Files

- **`components/admin/ImageUrlInput.tsx`** — Switched from `uploadImage` server action to `uploadImageDirect` client helper.
- **`components/admin/ImagePairEditor.tsx`** — Same switch to `uploadImageDirect`.
- **`messages/admin/en.ts`** — Updated format hint from "max 5 MB" to "max 50 MB".
- **`messages/admin/zh.ts`** — Updated format hint from "最大 5 MB" to "最大 50 MB".

### Documentation

- **`CLAUDE.md`** — Updated Upload action entry to describe presigned URL flow; added `upload-constants.ts` and `upload-client.ts` to project structure; updated S3 client description.
- **`docs/architecture.md`** — Rewrote Image Upload section with architecture diagram, new subsections for Upload Constants, Presign API Route, and Client Helper; added new files to directory listing.
- **`docs/README.md`** — Updated admin dashboard description to mention presigned S3 URL uploads (50 MB images, 1 GB batch ZIP).

## Architecture

```
Before:  Browser → FormData → Server Action (5 MB buffer) → S3
After:   Browser → JSON metadata → API Route → presigned URL → Browser PUT → S3 (50 MB)
```

All admin upload paths now use presigned S3 URLs:
- `ImageUrlInput` → `uploadImageDirect()` → presigned S3 PUT
- `ImagePairEditor` → `uploadImageDirect()` → presigned S3 PUT
- Batch upload → already used presigned S3 URLs (unchanged)
- Batch processor → server-to-server S3 operations (unchanged)
