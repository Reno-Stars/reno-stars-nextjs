## Social Media Posts Admin Feature

New admin content type for managing social media campaigns. Each campaign row contains content for all 3 platforms (Instagram, Facebook, Xiaohongshu). AI generation via GPT-4o populates all platform fields from a linked blog post, project, or site.

### Schema

New `social_post_status` enum (`draft`, `ready`, `published`) and `social_media_posts` table:

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `title_en` / `title_zh` | VARCHAR(200) | Campaign label (bilingual) |
| `instagram_caption_en` / `_zh` | TEXT | Instagram captions |
| `instagram_hashtags_en` / `_zh` | TEXT | Instagram hashtags |
| `facebook_caption_en` / `_zh` | TEXT | Facebook captions |
| `facebook_hashtags_en` / `_zh` | TEXT | Facebook hashtags |
| `xiaohongshu_caption_zh` / `_en` | TEXT | Xiaohongshu captions (ZH primary) |
| `xiaohongshu_topic_tags_zh` | TEXT | Xiaohongshu topic tags |
| `selected_image_urls` | JSONB | Array of selected source image URLs |
| `blog_post_id` / `project_id` / `site_id` | UUID (nullable FK) | Source reference (at most one via CHECK constraint) |
| `status` | `social_post_status` | Draft / Ready / Published |
| `scheduled_at` / `published_at` | TIMESTAMP | Scheduling and publish tracking |
| `notes` | TEXT | Internal notes |
| `created_at` / `updated_at` | TIMESTAMP | Timestamps |

Indexes on `status`, `blog_post_id`, `project_id`, `site_id`, `scheduled_at`, `created_at`. CHECK constraint ensures at most one source FK is non-null. All FKs use `onDelete: 'set null'`.

### AI Generation (`lib/ai/social-post-generator.ts`)

- System prompt with platform-specific formatting rules (Instagram: emoji-rich, 2200 char max, 30 hashtags; Facebook: narrative, 3-5 hashtags; Xiaohongshu: Chinese-first, `#topic tags#` format)
- Zod-validated response schema (`SocialPostGenerationSchema`) with `.min(1)` on all 13 string fields
- Reuses `ProjectDataForBlog` and `SiteDataForBlog` types from `blog-generator.ts`
- Three entry points: `generateSocialPostsFromBlog()`, `generateSocialPostsFromProject()`, `generateSocialPostsFromSite()`
- Uses `AI_CONFIG.maxTokensSocialPost: 4096`

### Server Actions (`app/actions/admin/social-posts.ts`)

- `createSocialMediaPost()` / `updateSocialMediaPost()` / `deleteSocialMediaPost()` — standard CRUD with auth, UUID validation, text length validation
- `updateSocialPostStatus()` — inline status toggle from list page
- `generateSocialPostFromBlog()` / `generateSocialPostFromProject()` / `generateSocialPostFromSite()` — AI generation returning content for form population (no DB insert)
- `getSourceImages()` — fetches available images from source (hero, image pairs, child project images) for the image picker
- Uses shared `groupBy()` helper (exported from `queries.ts`) for grouping child project relations
- Type-safe update pattern with conditional spread for `publishedAt` preservation
- Runtime validation of `sourceType` and `status` values at server action boundary
- Delete verification via `.returning()` check

### Admin UI

- **List page** (`social-posts/page.tsx`): Server component fetching via `getAllSocialMediaPostsAdmin()`
- **List client** (`SocialPostListClient.tsx`): DataTable with columns (Title, Source, Status, Scheduled, Date, Actions). `StatusCell` extracted as standalone component with own `useTransition` for optimistic status updates. Rollback on error.
- **Form** (`SocialPostForm.tsx`): Full campaign editor with:
  - Source selector (radio buttons: None/Blog/Project/Site + SearchableSelect dropdown)
  - AI Generate All button (fetches from source, populates all fields)
  - Platform tabs (Instagram / Facebook / Xiaohongshu) with WAI-ARIA Tabs pattern (ArrowLeft/ArrowRight keyboard navigation, `role="tablist"`, `aria-selected`, `tabIndex` roving, `hidden` attribute panels)
  - Image picker (grid of source images with toggle selection, `aria-pressed`, `aria-label`)
  - Campaign title (BilingualInput), status select, scheduling, notes
  - Standard hooks: `useActionState`, `useFormToast`, `useBeforeUnload`, `EditModeToggle`
- **Create page** (`new/page.tsx`): Fetches blog/project/site options via `Promise.all`
- **Edit page** (`[id]/page.tsx`): Runtime JSONB validation for `selectedImageUrls` (`Array.isArray` + `.filter` with type guard)
- **Sidebar**: Added under Content group (between Blog and Gallery)
- **Dashboard**: Added social posts count with `Megaphone` icon in Content section

## Miscellaneous Improvements

### Accessibility

- **Skip to content link** (`app/[locale]/layout.tsx`): Added `<a href="#main-content">` with `sr-only` / `focus:not-sr-only` styling. Bilingual text ("Skip to main content" / "跳到主要内容"). `<main id="main-content">` wraps page content.
- **Navbar Escape key** (`components/Navbar.tsx`): Areas dropdown now closes on Escape key press (desktop).
- **Reduced motion** (`app/globals.css`, `app/admin/admin-responsive.css`): Added `@media (prefers-reduced-motion: reduce)` overrides for `modalFadeIn` and `admin-spin` keyframes.
- **Admin auth error boundary** (`app/admin/(auth)/error.tsx`): New error boundary for admin auth routes with `role="alert"`, bilingual via `useAdminTranslations()`.

### SEO

- **ProjectCategorySchema** (`components/structured-data/ProjectCategorySchema.tsx`): New JSON-LD `ItemList` schema for project category pages. Lists all projects in the category with position and URL.
- **Blog pagination links** (`app/[locale]/blog/page.tsx`): Added `<link rel="prev">` / `<link rel="next">` for paginated blog listing.

### Performance

- **Homepage lazy loading** (`components/pages/HomePage.tsx`): `FaqSection` and `ContactSection` now use `next/dynamic` for code-splitting (below-fold client components).

### Design

- **TEXT_MUTED opacity** (`lib/theme.ts`): Changed from `0.5` to `0.6` for better contrast.

### Upload error handling

- **Upload action** (`app/actions/admin/upload.ts`): Wrapped S3 `PutObjectCommand` in try/catch for better error reporting.
