## SEO Character Counters (`components/admin/BilingualInput.tsx`)

### New `maxLength` Prop

Added optional `maxLength` prop to `BilingualInput`. When provided, renders a `{length}/{max}` counter below each EN/ZH input with color-coded feedback:

```typescript
function getCounterColor(length: number, max: number): string {
  if (length >= max) return ERROR;      // Red at limit
  if (length >= max * 0.8) return GOLD; // Gold 80-100%
  return TEXT_MUTED;                    // Gray default
}
```

### Dual-Mode Length Tracking

- **Controlled mode** (valueEn/onChangeEn): Derives length from the controlled value directly
- **Uncontrolled mode** (defaultValueEn): Tracks length via `useState` + `onInput` handler, only attached when `maxLength` is set

### SEO Fields Updated

Both `ProjectForm` and `SiteForm` now pass `maxLength` to their SEO BilingualInputs using schema constants:

| Field | Constant | Max Length |
|-------|----------|------------|
| Meta Title | `SEO_META_TITLE_MAX` | 70 |
| Meta Description | `SEO_META_DESCRIPTION_MAX` | 155 |
| Focus Keyword | `SEO_FOCUS_KEYWORD_MAX` | 50 |
| SEO Keywords | *(none)* | *(unbounded)* |

## Slug Auto-Rename Warning

### Server Action Changes (`projects.ts`, `sites.ts`)

`updateProject` and `updateSite` now capture the user-submitted slug before calling `ensureUniqueSlug()` and compare afterward:

```typescript
const submittedSlug = data.slug;
// ... ensureUniqueSlug() call ...
const renamedSlug = data.slug !== submittedSlug ? data.slug : undefined;

return { success: true, ...(renamedSlug ? { renamedSlug } : {}) };
```

Return type widened to `{ success?: boolean; error?: string; renamedSlug?: string }`.

### Warning Toast Type (`ToastProvider.tsx`)

Added `'warning'` as a third toast type alongside `'success'` and `'error'`:

```typescript
type ToastType = 'success' | 'error' | 'warning';

const TOAST_COLORS: Record<ToastType, { bg: string; fg: string }> = {
  success: { bg: SUCCESS_BG, fg: SUCCESS },
  error: { bg: ERROR_BG, fg: ERROR },
  warning: { bg: GOLD_PALE, fg: GOLD },
};
```

Refactored inline ternary color logic to use `TOAST_COLORS` map for all three types.

### `useFormToast` Enhancement (`useFormToast.ts`)

Extended to detect `renamedSlug` in state and show a warning toast:

```typescript
export function useFormToast(
  state: { success?: boolean; error?: string; renamedSlug?: string },
  successMessage: string,
  options?: { slugRenameLabel?: string }
)
```

- Accepts optional `slugRenameLabel` for i18n support
- Falls back to English `'Slug auto-adjusted to'` for callers that don't pass a label
- ProjectForm and SiteForm pass `t.common.slugRenamed`

### Translation Keys

Added `common.slugRenamed` to admin translations:
- **EN**: `"Slug auto-adjusted to"`
- **ZH**: `"Slug 已自动调整为"`

## Session 2: SEO Optimization Fixes

Comprehensive SEO audit identified issues across structured data, image alt text, and meta descriptions. All fixes applied and verified.

### ProjectSchema Restructured (`components/structured-data/ProjectSchema.tsx`)

Changed `@type` from `CreativeWork` to `WebPage` with `mainEntity: Service`. This properly tells search engines "this page documents a completed renovation service":

```typescript
const schema = {
  '@context': 'https://schema.org',
  '@type': 'WebPage',
  name, description, url,
  image: allImages,           // Flat URL strings (Google-preferred)
  mainEntity: {
    '@type': 'Service',
    name, description,
    provider: {
      '@type': 'HomeAndConstructionBusiness',
      // ... company info + aggregateRating
    },
    serviceType,              // e.g., "Kitchen Renovation"
    areaServed: { ... },      // Location with PostalAddress
  },
};
```

Key changes:
- Outer `WebPage` uses flat URL strings for images (Google Rich Results preference)
- `mainEntity` is a `Service` with the `HomeAndConstructionBusiness` provider
- `areaServed` replaces `contentLocation` for better geo-targeting
- `aggregateRating` moved from standalone `provider` to nested provider object

### ArticleSchema ImageObject Dimensions (`components/structured-data/ArticleSchema.tsx`)

Added `width: 1200` and `height: 630` to the `ImageObject` in ArticleSchema. These match the standard OG image dimensions and help Google display images correctly in rich snippets.

### Bilingual Image Alt Text Fallbacks

Replaced 12 generic `'Before'`/`'After'` alt text fallbacks across 3 components with descriptive, bilingual alternatives using project/site title + i18n translation keys:

**Before:**
```tsx
alt={pair.beforeImage.alt || 'Before'}
alt={pair.afterImage.alt || 'After'}
```

**After:**
```tsx
alt={pair.beforeImage.alt || `${title} - ${t('projects.beforeLabel')}`}
alt={pair.afterImage.alt || `${title} - ${t('projects.afterLabel')}`}
```

This renders as:
- EN: `"Kitchen Remodel - Before"` / `"Kitchen Remodel - After"`
- ZH: `"厨房改造 - 施工前"` / `"厨房改造 - 施工后"`

Files updated: `ProjectModal.tsx` (2 instances), `ProjectDetailPage.tsx` (4 instances), `SiteDetailPage.tsx` (4 instances).
