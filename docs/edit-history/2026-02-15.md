## Process Page PDF Download (`components/pages/ProcessPage.tsx`)

### New PDF Download Handler

Added `handlePdfDownload` callback using `jsPDF`:

```typescript
const handlePdfDownload = useCallback(async () => {
  const dataUrl = await capturePoster();

  const img = new Image();
  await new Promise<void>((resolve, reject) => {
    img.onload = () => resolve();
    img.onerror = () => reject(new Error('Failed to load image'));
    img.src = dataUrl;  // Set AFTER listeners to prevent cache race
  });

  const imgWidth = img.naturalWidth;
  const imgHeight = img.naturalHeight;
  const orientation = imgWidth > imgHeight ? 'landscape' : 'portrait';
  const pdf = new jsPDF({ orientation, unit: 'px', format: [imgWidth, imgHeight] });
  pdf.addImage(dataUrl, 'PNG', 0, 0, imgWidth, imgHeight);
  pdf.save('reno-stars-process-poster.pdf');
}, [isPdfDownloading, capturePoster]);
```

Key design decisions:
- Page size set dynamically from captured image dimensions (not fixed A4) to avoid cropping
- `jsPDF.addImage()` accepts data URL directly — no extra conversion needed
- PDF button hidden on mobile via `hidden sm:inline-flex`
- Independent `isPdfDownloading` + `pdfDownloadStatus` state for each button

### Shared `capturePoster()` Helper

Extracted duplicate `toPng` call into reusable callback:

```typescript
const capturePoster = useCallback(() => {
  if (!posterRef.current) return Promise.reject(new Error('Poster ref not available'));
  return toPng(posterRef.current, { pixelRatio: 2, backgroundColor: SURFACE });
}, []);
```

Used by both `handleDownload` (PNG) and `handlePdfDownload` (PDF).

### Image Race Condition Fix

Fixed `new Image()` pattern where `src` was set before `onload`/`onerror` listeners, which could miss cached images:

```typescript
// Before (race condition):
const img = new Image();
img.src = dataUrl;
await new Promise<void>((resolve, reject) => {
  img.onload = () => resolve();
  img.onerror = () => reject(...);
});

// After (safe):
const img = new Image();
await new Promise<void>((resolve, reject) => {
  img.onload = () => resolve();
  img.onerror = () => reject(...);
  img.src = dataUrl;  // Set AFTER listeners
});
```

### setTimeout Cleanup on Unmount

Added timer refs and cleanup effect to prevent setState on unmounted component:

```typescript
const statusTimerRef = useRef<ReturnType<typeof setTimeout>>(undefined);
const pdfStatusTimerRef = useRef<ReturnType<typeof setTimeout>>(undefined);

useEffect(() => {
  return () => {
    clearTimeout(statusTimerRef.current);
    clearTimeout(pdfStatusTimerRef.current);
  };
}, []);
```

All `setTimeout` calls now store their IDs: `statusTimerRef.current = setTimeout(...)`.
