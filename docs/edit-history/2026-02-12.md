
## Image Pairs Schema Reference

### `project_image_pairs` Table

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `project_id` | UUID | FK to projects (cascade delete) |
| `before_image_url` | VARCHAR(500) | Before image URL (optional) |
| `before_alt_text_en` | VARCHAR(255) | English alt text for before |
| `before_alt_text_zh` | VARCHAR(255) | Chinese alt text for before |
| `after_image_url` | VARCHAR(500) | After image URL (optional) |
| `after_alt_text_en` | VARCHAR(255) | English alt text for after |
| `after_alt_text_zh` | VARCHAR(255) | Chinese alt text for after |
| `title_en` | VARCHAR(200) | English title for SEO |
| `title_zh` | VARCHAR(200) | Chinese title for SEO |
| `caption_en` | TEXT | English caption |
| `caption_zh` | TEXT | Chinese caption |
| `photographer_credit` | VARCHAR(100) | Photographer attribution |
| `keywords` | TEXT | SEO keywords (comma-separated) |
| `display_order` | INTEGER | Sort order (default 0) |
| `created_at` | TIMESTAMP | Creation timestamp |

**Constraint:** At least one of `before_image_url` or `after_image_url` must be non-null.

### `site_image_pairs` Table

Same structure with `site_id` FK instead of `project_id`.

## SEO-Friendly Image Upload Naming (Session 2)

### Upload Action (`app/actions/admin/upload.ts`)

Added `customKey` support to `uploadImage()` server action:
- Accepts optional `customKey` field from FormData
- If provided, uses `uploads/admin/{customKey}.{ext}` as S3 key
- Falls back to timestamp-random naming when empty
- **Sanitization:** `customKey` stripped to `/[a-z0-9-]/` only, capped at 200 chars

```typescript
// Before: uploads/admin/1739370000000-3a4d9c2b.jpg
// After:  uploads/admin/richmond-kitchen-hero.jpg
```

### ImagePairEditor (`components/admin/ImagePairEditor.tsx`)

New features:
- **`slug` prop** — Passed from ProjectForm/SiteForm
- **`humanizeSlug()` helper** — Converts `richmond-kitchen` to `Richmond Kitchen`
- **Custom S3 keys** — Before/after uploads use `{slug}-before-renovation-{index}` / `{slug}-after-renovation-{index}`
- **Auto-fill alt text** — On upload, auto-populates empty alt fields:
  - EN: `Richmond Kitchen - Before Renovation 1`
  - ZH: `Richmond Kitchen - 装修前 1`
- **`pairsRef` pattern** — Uses `useRef` + `useEffect` sync to read latest `pairs` state after async upload gap (avoids stale closure). This keeps `onChange` as simple `(pairs: ImagePairEntry[]) => void` instead of requiring functional updater.

### ImageUrlInput (`components/admin/ImageUrlInput.tsx`)

New props:
- **`slug?: string`** — When provided, builds custom S3 key
- **`imageRole?: string`** (default `'hero'`) — S3 key suffix

Upload key format: `{slug}-{imageRole}.{ext}` (e.g., `richmond-kitchen-hero.webp`)

### ProjectForm & SiteForm

Both forms updated:
- Added controlled `slug` state (synced with `initialData` via `useEffect`)
- Slug `<input>` converted from `defaultValue` to `value` + `onChange`
- `slug` prop passed to both `<ImageUrlInput>` and `<ImagePairEditor>`

### Edge Cases

- **Empty slug** — Falls back to timestamp-random naming (no `customKey` sent)
- **S3 collisions** — Re-uploading to the same slot overwrites (acceptable)
- **Existing images** — Not affected; only applies to new uploads
- **Alt text preservation** — Only auto-fills when field is currently empty
