### SEO Fixes

#### Double-Slash URL Bug (`lib/utils.ts`)

- **Root cause:** `NEXT_PUBLIC_BASE_URL` env var had a trailing slash, causing every canonical, hreflang, sitemap, and robots.txt URL to contain `//` (e.g., `reno-stars.com//en/`).
- **Fix:** `getBaseUrl()` now strips trailing slashes: `url.replace(/\/+$/, '')`.
- Affected: `app/sitemap.ts`, `app/robots.ts`, all `generateMetadata` functions via `buildAlternates()`.

### Hydration Mismatch Fix (Layout Restructure)

#### Problem

Root `app/not-found.tsx` rendered its own `<html>/<body>` tags. When a 404 occurred within the `[locale]` layout (which also rendered `<html>/<body>`), nested `<html>` tags caused React hydration mismatch: server rendered `lang="zh"` but client expected `lang="en"`.

#### Solution — Single `<html>/<body>` in root layout only

- **`app/layout.tsx`** — Now provides `<html lang={locale}><body>` using `getLocale()` from next-intl/server (previously just returned `children`)
- **`app/[locale]/layout.tsx`** — Removed `<html>`, `<head>`, `<body>` tags. Renders `NextIntlClientProvider` + `LocalBusinessSchema` + `Navbar` + `Footer` directly
- **`app/not-found.tsx`** — Removed `<html>/<body>` tags. Just renders 404 content div
- **`app/[locale]/not-found.tsx`** — New file. Locale-aware 404 page using `getTranslations('common')`, renders inside locale layout (no `<html>/<body>`)

### Middleware → Proxy Migration

#### Problem

Next.js 16 deprecated `middleware.ts` in favor of `proxy.ts`. Both files existed, causing a startup crash: "Both middleware file and proxy file are detected."

#### Solution

- **Merged `middleware.ts` into `proxy.ts`** — Combined admin auth (session cookie check, redirect to `/admin/login`) with i18n routing (next-intl middleware) and security headers (CSP, X-Frame-Options, etc.)
- **Deleted `middleware.ts`**
- All responses now get security headers via `addSecurityHeaders()` helper
- Uses `routing` from `i18n/config` (consistent with middleware.ts) instead of individual locale params

### Blog Crawler (`scripts/seed-blog.ts`)

New script that crawls the old WordPress site for blog content:

- **English**: Fetched via WP REST API (`/wp-json/wp/v2/renovation_article`) — structured JSON
- **Chinese**: Crawled from `/zh/renovation_article/{slug}/` HTML pages (WPML doesn't expose ZH via REST API)
- **Title extraction**: Tries `og:title`, `<title>`, `<h1>` tags; prefers whichever contains Chinese characters
- **Content cleaning**: Strips scripts, styles, shortcodes, inline styles, data attributes
- **Idempotent**: Uses `onConflictDoUpdate` to refresh content on re-run
- **Result**: 22 articles seeded, 14/22 with Chinese titles, all with distinct ZH body content where translations exist
- **Command**: `pnpm db:seed:blog`
