# 2026-03-11 — Batch Upload Standalone Mode + AI PO Number Extraction

## Summary

Added standalone projects mode to batch upload, enabling ZIP uploads that create individual projects (without site wrappers) under the standalone site container. Moved PO number, budget, duration, and excerpt extraction from hard regex to AI-generated fields across both batch upload and admin form UI. Refactored batch processor with shared helpers and early branching for cleaner code.

## Changes

### Batch Upload — Standalone Mode

- **`lib/batch/types.ts`** — Added `BatchUploadMode` type (`'sites' | 'standalone'`), `ParsedStandaloneStructure` interface.
- **`lib/batch/zip-parser.ts`** — Extracted shared `extractFilesFromZip()` from `parseZip()`. Added `parseZipStandalone()` that flattens subfolders into top-level projects. Subfolder notes/products intentionally ignored.
- **`lib/batch/batch-processor.ts`** — Added standalone mode processor (`processStandaloneMode()`). Early branching based on `options.mode` eliminates dual-nullable pattern. Looks up `STANDALONE_SITE_SLUG` container, queries `max(display_order_in_site)`, inserts projects sequentially. Extracted shared helpers: `collectProjectImages()`, `collectAfterImageUrls()`, `generateBlogsForEntities()`, `finalizeJob()`, `cleanupTempZip()`. Standalone blog generation uses `generateBlogFromProject()`.
- **`app/admin/(dashboard)/batch-upload/BatchUploadClient.tsx`** — Added mode tab bar (Sites / Standalone Projects) with ARIA tab pattern. Mode-adaptive UI: description text, folder structure help, example ZIP download link, blog generation help text, results summary cards, and action buttons all adapt to selected mode. Extracted `actionLinkStyle` constant for deduplicated link styles. Fixed error list `key={i}` → `key={\`${idx}-${err}\`}` for uniqueness.
- **`app/admin/(dashboard)/batch-upload/api/route.ts`** — Added `mode` to request body parsing and `BatchJobOptions` insertion.
- **`lib/db/schema.ts`** — Added `mode: 'sites' | 'standalone'` to `BatchJobOptions` interface.
- **`messages/admin/en.ts`** — Added batch upload standalone mode i18n keys: `tabSites`, `tabStandalone`, `descriptionStandalone`, `generateBlogHelpStandalone`, `folderStructureHelpStandalone`, `reviewProjects`.
- **`messages/admin/zh.ts`** — Added corresponding Chinese translations.

### AI PO Number Extraction (replaces hard regex)

- **`lib/ai/content-optimizer.ts`** — Added `poNumber` field to both `ProjectDescriptionSchema` and `SiteDescriptionSchema` Zod schemas. Added `poNumber` to both AI prompts with extraction guidelines (max 50 characters). Also added `budgetRange`, `durationEn/Zh`, `excerptEn/Zh` to `ProjectDescriptionSchema` (already existed on `SiteDescriptionSchema`).
- **`lib/admin/form-utils.ts`** — Removed `extractPoNumber()` function (replaced by AI extraction).
- **`components/admin/AIFieldGenerator.tsx`** — Removed `onRawNotes` prop and callback (no longer needed — PO comes via AI response).
- **`components/admin/AIProjectGenerator.tsx`** — Removed `onRawNotes` prop passthrough.
- **`components/admin/AISiteGenerator.tsx`** — Removed `onRawNotes` prop passthrough.
- **`components/admin/ProjectForm.tsx`** — Removed `handleRawNotes` callback and `extractPoNumber` import. Added controlled state for `poNumber`, `budgetMin`, `budgetMax`, `durationEn/Zh`, `excerptEn/Zh`. `handleAIGenerate` now populates all these fields from AI response (budget string parsed into min/max via regex). Added excerpt `BilingualTextarea` in SEO section. Added excerpt to pre-save warnings check.
- **`components/admin/SiteForm.tsx`** — Removed `handleRawNotes` callback and `extractPoNumber` import. Added controlled `poNumber` state. `handleAIGenerate` populates PO from AI response.
- **`lib/batch/batch-processor.ts`** — Removed `extractPoNumber` import and usage. Now uses `siteData.poNumber` and `projectData.poNumber` from AI response. Added `poNumber: ''` to both `fallbackSiteData()` and `fallbackProjectData()`.

### macOS ZIP Artifact Filtering

- **`lib/batch/zip-parser.ts`** — Added `filename.startsWith('._')` guard in `extractFilesFromZip()` to skip macOS resource fork artifacts (`._After 1.jpg`, `._notes.txt`, etc.) that appear alongside real files in ZIPs created on macOS. Existing `__MACOSX` folder filter was insufficient for `._` prefixed files at the same level as real content.

### Standalone Projects Sort Order

- **`lib/db/queries.ts`** — Added `createdAt` to `ProjectSummary` interface and `getAllProjectsBySiteAdmin()` select.
- **`app/admin/(dashboard)/sites/SitesListClient.tsx`** — Added `createdAt` to `StandaloneProjectRow` interface. Standalone projects memo now sorts by `createdAt` descending (newest first).

### Standalone Site Auto-Creation (`ensureStandaloneSite`)

- **`lib/db/queries.ts`** — Added `ensureStandaloneSite()` shared find-or-create function for the standalone projects container site. Returns site ID, creating the container if it doesn't exist (slug: `STANDALONE_SITE_SLUG`, unpublished, hidden). Replaces the previous throw-on-missing pattern.
- **`lib/batch/batch-processor.ts`** — Standalone mode now uses shared `ensureStandaloneSite()` instead of inline find-or-throw. Removed `STANDALONE_SITE_SLUG` import (no longer needed directly).
- **`app/admin/(dashboard)/sites/page.tsx`** — Now calls `ensureStandaloneSite()` in `Promise.all` alongside data fetching, guaranteeing the "New Standalone Project" button always works even on a fresh database.

### Hide Standalone Container from Site Lists

- **`app/admin/(dashboard)/sites/SitesListClient.tsx`** — Added `visibleSites` memo that filters out the standalone container site by `standaloneSiteId`. Used by Sites tab and Whole House tab DataTables. The `standaloneProjects` memo still uses the unfiltered `sites` array (needs the container to collect its child projects via the "not shown as whole-house" filter).

### AI Service Type Detection + Descriptive Slugs

- **`lib/ai/content-optimizer.ts`** — Added `serviceType` field to `ProjectDescriptionSchema` (Zod enum: kitchen/bathroom/basement/cabinet/commercial, defaults to kitchen). Updated AI prompt with service type detection guidelines (infer from notes content — e.g., 卫生间 → bathroom). Updated slug guidelines to require specific, descriptive slugs (scope, style, materials, location). Updated title guidelines to reflect specific scope instead of generic labels. Added `serviceType` to JSON response format example.
- **`lib/batch/batch-processor.ts`** — Fixed AI prompt bias: when notes exist, prompt now says `Renovation project:` instead of `${category.en} renovation project:` so AI isn't biased by folder-name heuristic. `saveProject()` now prefers AI-detected `serviceType` over `parsedProject.serviceType` from folder detection. Added `serviceType` to `fallbackProjectData()`.
- **`components/admin/ProjectForm.tsx`** — `handleAIGenerate` now sets the service type dropdown from AI response (`data.serviceType`).
