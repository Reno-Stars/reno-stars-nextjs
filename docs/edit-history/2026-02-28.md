# Edit History: 2026-02-28

## Summary

**Session 1** — Batch Upload feature: new admin page for uploading ZIP files of renovation images that auto-creates sites, projects, image pairs, and optionally blog posts via AI. Includes ZIP parsing, S3 upload, AI metadata generation, and progress tracking via polling.

**Session 2** — Code review of the batch upload feature identified 18 issues (4 critical, 8 warnings, 6 suggestions). All 18 fixed:

### Fixes Applied

1. **Critical: Fire-and-forget async processing** — Replaced `processBatchUpload().catch()` with `after()` from `next/server` in the process route handler, ensuring the function lifetime extends beyond the response on Vercel.
2. **Critical: S3 client created per image upload** — Cached the S3 client as a module-level singleton in `lib/admin/s3.ts` using `undefined` sentinel to distinguish "not yet initialized" from "missing credentials".
3. **Critical: N+1 AI alt text API calls** — Alt text generation parallelized via `generateAltTextsInBatches()` with `ALT_TEXT_BATCH_SIZE = 5` using `Promise.allSettled`. Pre-generates all alt text before DB insertion.
4. **Critical: test-data/ not in .gitignore** — Added `test-data/` to `.gitignore`.
5. **Warning: Hardcoded S3_BUCKET fallback** — Extracted `S3_BUCKET` constant to `lib/admin/s3.ts`, used everywhere instead of inline `process.env.S3_BUCKET || 'reno-stars'`.
6. **Warning: `updateJob` status typed as `string`** — Changed to `BatchJobStatus` type from schema.
7. **Warning: Hardcoded English error messages in client** — Added `errorZipOnly`, `errorTooLarge`, `errorS3NotConfigured`, `errorUploadFailed` translation keys to both `en.ts` and `zh.ts`. Updated `handleFile` and `handleSubmit` in `BatchUploadClient`.
8. **Warning: `JobStatus` type duplicated from schema** — Exported `BATCH_JOB_STATUSES` const array, `BatchJobStatus` type, and `BatchJobOptions` interface from `lib/db/schema.ts`. Client component imports `BatchJobStatus` from schema.
9. **Warning: `unzipSync` blocks event loop** — Replaced synchronous `unzipSync` with async `unzip` callback wrapped in a Promise. `parseZip()` is now `async`.
10. **Warning: No orphan site cleanup** — Added `cleanupOrphanedSite()` that deletes sites with 0 child projects on failure. Tracks `projectsCreatedForSite` counter and cleans up if all projects fail.
11. **Warning: Only first site slug used for all S3 uploads** — Moved slug prefix computation inside the per-site loop so each site's images get its own prefix.
12. **Suggestion: Dynamic imports of S3 commands** — Moved `GetObjectCommand` and `DeleteObjectCommand` to static top-level imports.
13. **Suggestion: Steps 4/5 not distinct** — Restructured batch processor so AI metadata generation genuinely runs in a separate pass (step 4) before DB insertion (step 5).
14. **Suggestion: JSON stored as `text` columns** — Changed `options`, `createdSiteIds`, `createdProjectIds`, `createdBlogPostIds`, and `errors` columns from `text` to `jsonb` with Drizzle `$type<>()` annotations. Eliminated all `JSON.parse`/`JSON.stringify` calls.

### Schema Changes

New table `batch_upload_jobs` with `batch_job_status` enum:

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `status` | `batch_job_status` enum | pending → extracting → uploading → generating → saving → generating_blog → completed/failed/partial |
| `file_name` | VARCHAR(255) | Original ZIP filename |
| `file_size_bytes` | INTEGER | File size |
| `options` | JSONB | `{ generateBlog: boolean }` |
| `total_images` | INTEGER | Total image count from ZIP |
| `processed_images` | INTEGER | Upload progress counter |
| `current_step_label` | VARCHAR(200) | Human-readable status message |
| `created_site_ids` | JSONB | Array of created site UUIDs |
| `created_project_ids` | JSONB | Array of created project UUIDs |
| `created_blog_post_ids` | JSONB | Array of created blog post UUIDs |
| `errors` | JSONB | Array of error messages |
| `started_at` / `completed_at` | TIMESTAMP | Timing |
| `created_at` | TIMESTAMP | Row creation time |

Indexed on `status`.
