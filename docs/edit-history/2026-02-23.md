# Edit History: 2026-02-23

## Summary

**Session 7** — Admin UX audit and fixes. A comprehensive PM-level UX review of the admin panel identified 8 issues across bilingual consistency, workflow improvements, and visual polish. All 8 were implemented and verified.

### Changes Made

1. **Bilingual error/loading pages**: `error.tsx` and `loading.tsx` now use `useAdminTranslations()` instead of hardcoded English. Loading page redesigned with animated CSS spinner.
2. **ToggleButton bilingual defaults**: Removed hardcoded `'Yes'`/`'No'` defaults; now uses `t.common.yes`/`t.common.no` from admin translations for proper EN/ZH switching.
3. **Contact status label translations**: Filter buttons and status dropdown in `ContactsListClient` now use `t.status[s]` instead of raw English enum values (e.g., `'new'` → `'New'`/`'新的'`).
4. **Back navigation on edit/new pages**: Added `backHref` and `backLabelKey` props to `AdminPageHeader`. Applied to 14 edit/new page files across blog, FAQs, gallery, services, partners, service areas, trust badges, and social links.
5. **Standardized toggle buttons (blog + sites)**: Replaced raw `<button>` publish toggles in `BlogListClient` and `SitesListClient` with the shared `ToggleButton` component, adding `pendingId` state for loading feedback.
6. **Dashboard pre-filter for new contacts**: Changed "New Contacts" card href from `/admin/contacts` to `/admin/contacts?status=new`. Contacts page reads `?status` query param via `useSearchParams` to set the initial filter.
7. **Delete button color standardization**: Changed delete buttons from gray (`TEXT_MID`) to red (`ERROR`) across all 6 list pages: gallery, service areas, trust badges, FAQs, services, and partners.
8. **Loading spinner**: Added animated gold CSS spinner to admin loading page (done as part of fix #1).
