## Gallery — Create & Delete

Added full create and delete CRUD to the Gallery admin panel, matching the established FAQs pattern.

### Server Actions (`app/actions/admin/gallery.ts`)

**`createGalleryItem()`** — New action:
- Validates image URL (required, valid format), category, title lengths, display order
- Inserts via `db.insert(galleryItems).values(...)`
- Redirects to `/admin/gallery` after creation (uses `redirect()` outside try/catch)

**`deleteGalleryItem()`** — New action:
- Validates UUID, deletes with `.returning()` check
- No FK constraints to check (gallery items are standalone)

### Form (`GalleryItemForm.tsx`)

Added `isNew` prop support:
- `initialData` now optional, with sensible defaults (kitchen category, published by default)
- `EditModeToggle` hidden when `isNew`
- Submit button label switches between create/update
- `useEffect` for category sync guards against undefined `initialData`

### List Client (`GalleryListClient.tsx`)

Major enhancements:
- **Delete button + ConfirmDialog** — Matching FAQs pattern
- **Tetris Preview Grid** — Visual preview of published gallery items using the same layout pattern as the frontend `TetrisGallery.tsx` (12-item repeating pattern with `col-span-2` for wide items)
- **Category Filter Buttons** — Filter the preview grid by gallery category (All / Kitchen / Bathroom / Whole House / Commercial)

### New Page (`app/admin/(dashboard)/gallery/new/page.tsx`)

Standard create page with `AdminPageHeader` + `GalleryItemForm` with `isNew` prop.

## Service Areas — Create & Delete

Added full create and delete CRUD to the Service Areas admin panel with FK safety.

### Server Actions (`app/actions/admin/service-areas.ts`)

**`createServiceArea()`** — New action:
- Validates slug (required, `isValidSlug()`), names (required), descriptions, display order
- Uses `ensureUniqueSlug()` for collision safety (queries existing slugs first)
- Inserts via `db.insert(serviceAreas).values(...)`

**`deleteServiceArea()`** — New action:
- **FK Safety Check**: Queries `contact_submissions` table for rows referencing this area via `preferredAreaId`
- If any contacts reference the area, returns error: `"Cannot delete: X contact(s) reference this area."`
- Only deletes if no FK references exist

### Form (`ServiceAreaForm.tsx`)

Added `isNew` prop support:
- `initialData` now optional, with sensible defaults
- Slug field: editable `<input name="slug">` in create mode, read-only display-only in edit mode
- `EditModeToggle` hidden when `isNew`
- Submit button label: `createServiceArea` vs `update`

### List Client (`ServiceAreasListClient.tsx`)

- Delete button + `ConfirmDialog` matching FAQs pattern
- Shared `useTransition` between toggle and delete (matches existing pattern)

### List Page (`page.tsx`)

Added `actionKey` and `actionHref` to `AdminPageHeader` for the "Add Service Area" button.

### New Page (`app/admin/(dashboard)/service-areas/new/page.tsx`)

Standard create page with `AdminPageHeader` + `ServiceAreaForm` with `isNew` prop.

## DataTable Enhancement

### `headerAction` Prop (`components/admin/DataTable.tsx`)

New optional `headerAction: ReactNode` prop renders content on the right side of the search bar row. The search bar and header action are wrapped in a flexbox with `justify-content: space-between`.

Used by the Gallery list client for category filter buttons.

---

## Upload Improvements (`app/actions/admin/upload.ts`)

- **`S3_PUBLIC_URL` env var**: New environment variable for the public-facing URL of the S3 bucket (e.g., R2 public bucket URL or MinIO public URL)
- Falls back to `NEXT_PUBLIC_STORAGE_PROVIDER` for backwards compatibility
- Now returns an error if neither `S3_PUBLIC_URL` nor `NEXT_PUBLIC_STORAGE_PROVIDER` is set (previously would return a relative path like `/gallery/image.jpg` which isn't valid)

---

## Form Utils (`lib/admin/form-utils.ts`)

`isValidUrl()` now accepts relative paths starting with `/` in addition to `http://` and `https://` URLs. This supports local/relative image paths.
