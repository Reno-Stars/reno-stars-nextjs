## Service Areas — Create & Delete

Added full create and delete CRUD to the Service Areas admin panel with FK safety.

### Server Actions (`app/actions/admin/service-areas.ts`)

**`createServiceArea()`** — New action:
- Validates slug (required, `isValidSlug()`), names (required), descriptions, display order
- Uses `ensureUniqueSlug()` for collision safety (queries existing slugs first)
- Inserts via `db.insert(serviceAreas).values(...)`

**`deleteServiceArea()`** — New action:
- **FK Safety Check**: Queries `contact_submissions` table for rows referencing this area via `preferredAreaId`
- If any contacts reference the area, returns error: `"Cannot delete: X contact(s) reference this area."`
- Only deletes if no FK references exist

## useDragReorder Hook

Extracted drag-and-drop reordering logic into a reusable hook at `hooks/useDragReorder.ts`.

### Features

- Generic type parameter `<T>` for item type
- Configurable callbacks: `getId`, `getDisplayOrder`, `isIncluded`, `onReorder`, `onSuccess`, `onError`
- Optimistic UI updates with server sync
- Proper cleanup with `mountedRef` pattern to prevent state updates after unmount
- Unhandled rejection handling with `.catch()`
- `shouldNavigate()` helper to distinguish clicks from drags

### Usage

```typescript
const {
  draggedId,
  dragOverId,
  localOrder,
  isReordering,
  handleDragStart,
  handleDragOver,
  handleDragLeave,
  handleDrop,
  handleDragEnd,
  shouldNavigate,
} = useDragReorder({
  items,
  getId: (item) => item.id,
  getDisplayOrder: (item) => item.displayOrder,
  isIncluded: (item) => item.isPublished,
  onReorder: reorderGalleryItems,
  onSuccess: () => router.refresh(),
  onError: (msg) => toast(msg, 'error'),
});
```

});

## Admin Panel Mobile Optimization

Added responsive mobile support to the entire admin dashboard. Uses a CSS-only override strategy (`admin-responsive.css`) with `className` attributes alongside existing inline styles — CSS `!important` at `@media (max-width: 768px)` overrides inline values. Zero visual change on desktop.

### New Files

| File | Description |
|------|-------------|
| `app/admin/admin-responsive.css` | Mobile-only CSS overrides (sidebar drawer, form grids, cards, page header, etc.) |
| `components/admin/DashboardShell.tsx` | Client component extracted from dashboard layout for sidebar state + mobile drawer |
| `hooks/useIsMobile.ts` | `matchMedia` hook returning boolean for mobile breakpoint detection |

### Architecture: DashboardShell

The dashboard layout (`app/admin/(dashboard)/layout.tsx`) is a server component (for `requireAuth()`). Client-side sidebar open/close state is handled by `DashboardShell`:
- Sidebar wrapper with `className="admin-sidebar"` + conditional `admin-sidebar--open`
- Overlay div (mobile only, `aria-hidden="true"`)
- Closes sidebar on: overlay click, Escape key, link navigation, resize to desktop
- Uses `useIsMobile()` to auto-close sidebar on desktop transition

### AdminLocaleProvider Changes

Added `sidebarOpen: boolean` + `setSidebarOpen` to the existing context (extends the interface). Context value is now memoized with `useMemo`.

### Sidebar Changes

- Added optional `onNavigate?: () => void` prop, called on each `<Link onClick={onNavigate}>`
- Changed `flexShrink: 0` → `flex: 1` so the nav fills its wrapper's full height (fixes white gap at bottom)

### TopBar Changes

- Added hamburger button (3-line SVG) with `className="admin-hamburger"` (CSS-hidden on desktop)
- Reads `setSidebarOpen` from context
- Translated `aria-label` (`t.topBar.openMenu`)
