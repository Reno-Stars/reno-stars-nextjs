# 2026-03-22

## Session 1: Client-Orchestrated Batch Upload (Fix Vercel Timeouts)

Rewrote batch upload from server-side pipeline to client-orchestrated flow. The old architecture uploaded the ZIP to S3 and processed everything (extraction, image upload, AI metadata, DB save, blog generation) in a single `after()` callback, which always exceeded Vercel's 60s function timeout. The new architecture moves heavy work to the browser and orchestrates small, fast server calls.

### Architecture Change

**Old:** Client uploads ZIP → single server call does everything → timeout
**New:** Client extracts ZIP locally → uploads images directly to S3 → calls server for AI/DB/blog in small batches → no single call exceeds ~15s

### New Files

- Client-side ZIP extraction using existing `fflate`-based `zip-parser.ts`
- Generates S3 keys client-side with timestamp + random component
