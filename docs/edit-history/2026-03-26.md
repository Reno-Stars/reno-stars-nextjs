### Batch delete & batch status update server actions
- `app/actions/admin/contacts.ts`: Added `batchDeleteContacts(ids)` and `batchUpdateContactStatus(ids, status)`. Both use `inArray()` for single-query batch operations. Validation: `requireAuth()`, UUID check on all IDs, `MAX_BATCH_SIZE = 100` limit, status enum check. Returns `{ deleted/updated: number }` on success.

### Contacts list batch selection UI
- `app/admin/(dashboard)/contacts/ContactsListClient.tsx`: Added checkbox column (`_select`) with per-row selection. "Select All" checkbox in `DataTable.headerAction`. Batch action bar appears when 1+ contacts selected, showing: selected count, select/deselect all toggle, batch status dropdown, and "Delete Selected" button (red). Batch delete shows `ConfirmDialog` with contact names listed via `items` prop.

### Selection state management
- `selectedIds` state tracks raw selection. `effectiveSelectedIds` (synchronous `useMemo`) prunes selection against current filter — prevents stale selections surviving filter changes. All UI and action handlers use `effectiveSelectedIds`. A `useEffect` syncs `selectedIds` state when contacts are removed from DB.
