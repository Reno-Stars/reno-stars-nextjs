# 2026-03-26 — Reviews Page, Whole House Guide, Code Review Fixes

## Summary

Added dedicated Reviews/Testimonials page (`/reviews/`) with Google Reviews integration, whole house renovation cost guide (`/guides/whole-house-renovation-cost-vancouver/`), and updated redirect rules for the new About page. Also performed comprehensive code review of the last 5 commits (About page, footer guides link, homepage gallery fix, process QR code, ISR revalidation), fixing 3 critical bugs, 10 warnings, and 8 suggestions across 11 files.

## New: Reviews/Testimonials Page

### Added `/reviews/` route with Google Reviews
- `app/[locale]/reviews/page.tsx`: New page route with `force-dynamic` rendering (bypasses stale pre-render cache). Fetches company data and Google Reviews. Renders `BreadcrumbSchema`, `FAQSchema`, `ReviewSchema` structured data. Full `generateMetadata()` with OG images and hreflang alternates.
- `components/pages/ReviewsPage.tsx`: Client component displaying Google Reviews with star ratings, relative timestamps (`Intl.RelativeTimeFormat`), reviewer avatars (via `GoogleAvatar`), summary stats, "Why Choose Us" section with trust signals, FAQ accordion, and CTA.
- `components/Navbar.tsx`: Added "Reviews" link to nav (now 10 links total).
- `components/Footer.tsx`: Added "Reviews" to quick links.
- `messages/en.json`, `messages/zh.json`: Added `metadata.reviews`, `reviewsPage`, and `nav.reviews` translation keys.

## Redirect Updates for About Page

### Removed blocking `/about` redirect
- `next.config.ts`: Removed `/:locale(en|zh)/about → /:locale/` redirect that was blocking the new About page route. Updated `/about-us` redirects to point to `/about` instead of homepage. Removed non-localized `/about → /en/` redirect.

## Critical: Neumorphic Shadow Bug in Cost Guide Pages

### `...neu` spread fixed — shadows now render
- `components/pages/BathroomCostGuidePage.tsx`: Replaced `...neu` (spreading a function) with `boxShadow: neu()` at 5 locations (stat cards, tier cards, factor cards, project links, tip cards). The `neu` export is an arrow function returning a box-shadow string — spreading it produced no CSS properties, so all cards were rendering flat without shadows.
- `components/pages/KitchenCostGuidePage.tsx`: Same fix at 5 locations.

### Stray icon imports removed from BathroomCostGuidePage
- `components/pages/BathroomCostGuidePage.tsx`: The `costTiers` mid-range entry had `ShowerHead, Wrench` as stray shorthand properties in the object literal (line 68). Removed them and cleaned up unused `ShowerHead`, `Wrench`, `Home` imports from lucide-react.

## Critical: XSS Sanitization in OrganizationSchema

### Added `.replace(/</g, '\\u003c')` to JSON-LD output
- `components/structured-data/OrganizationSchema.tsx`: Was the only structured-data component missing the standard `</script>` escape pattern. All other 12 components already applied this sanitization.

### Rewrote OrganizationSchema to use dynamic data
- Changed from hardcoded address (`21300 Gordon Way, Unit 188`) to `parseAddress(company.address)` — consistent with `LocalBusinessSchema`.
- Added optional `socialLinks` and `areas` props. `sameAs` now populated from social links (omitted when empty). `areaServed` now populated from DB areas (omitted when empty) — previously hardcoded 10 cities that were inconsistent with other components.
- Changed `description` from `company.tagline` (a short slogan) to a full descriptive string matching `LocalBusinessSchema`.
- Changed from named export to `export default` for consistency with all other structured-data components.

### Updated barrel export and about page
- `components/structured-data/index.ts`: Changed from `export { OrganizationSchema }` to `export { default as OrganizationSchema }`.
- `app/[locale]/about/page.tsx`: Import now uses barrel. Added `getSocialLinksFromDb()` and `getServiceAreasFromDb()` to `Promise.all`. Passes `socialLinks` and `areas` props to `OrganizationSchema`.

## AboutPage Improvements

### Data arrays hoisted and memoized
- `components/pages/AboutPage.tsx`: `VALUES` and `SERVICES` arrays hoisted to module scope (static, no runtime deps). `stats` array wrapped in `useMemo([company])`. Added `useMemo` import.

- `ABOUT_FAQ_KEYS`, `SERVICE_AREA_CITIES` constants defined at module scope in `AboutPage.tsx`. FAQ section and city list now reference these constants instead of inline arrays. `app/[locale]/about/page.tsx` also uses a local `ABOUT_FAQ_KEYS` constant for programmatic FAQ generation.

## Footer Accessibility

## Navbar Accessibility

### Active page indication
- `components/Navbar.tsx`: Desktop and mobile nav links now include `aria-current="page"` when the current pathname matches the link href. Helps screen reader users identify the active page.

## ISR & Sitemap

### Added revalidate to sitemap
- `app/sitemap.ts`: Added `export const revalidate = 3600`. The sitemap runs 6 parallel DB queries on every request — without ISR, it contributed to excessive DB transfer on Neon's free tier.
