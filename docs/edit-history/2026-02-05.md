## Admin Locale Support

### New Files

- **`components/admin/AdminLocaleProvider.tsx`** — React context for admin locale state
  - Provides `locale` and `setLocale` via context
  - Persists preference in localStorage (`admin_locale` key)
  - Defaults to English on first visit
  - `useAdminLocale()` hook for consuming components

### Modified Files

- **`components/admin/TopBar.tsx`** — Added locale switcher buttons
  - EN/ZH toggle buttons with gold highlight for active locale
  - Uses `useAdminLocale()` hook

- **`app/admin/(dashboard)/layout.tsx`** — Wrapped with `AdminLocaleProvider`

- **List Clients Updated for Locale-Aware Display:**
  - `app/admin/(dashboard)/blog/BlogListClient.tsx` — titleEn/titleZh
  - `app/admin/(dashboard)/projects/ProjectsListClient.tsx` — titleEn/titleZh
  - `app/admin/(dashboard)/faqs/FaqsListClient.tsx` — questionEn/questionZh
  - `app/admin/(dashboard)/testimonials/TestimonialsListClient.tsx` — textEn/textZh
  - `app/admin/(dashboard)/gallery/GalleryListClient.tsx` — titleEn/titleZh
  - `app/admin/(dashboard)/service-areas/ServiceAreasListClient.tsx` — nameEn/nameZh
  - `app/admin/(dashboard)/trust-badges/TrustBadgesListClient.tsx` — badgeEn/badgeZh

### Pattern Used

All list clients follow this pattern:
\`\`\`tsx
const { locale } = useAdminLocale();

const columns = useMemo(() => {
  const getField = (row) => locale === 'zh' ? row.fieldZh : row.fieldEn;
  return [
    { key: locale === 'zh' ? 'fieldZh' : 'fieldEn', header: locale === 'zh' ? 'Field (ZH)' : 'Field (EN)' },
    // ...
  ];
}, [locale, ...]);
\`\`\`
