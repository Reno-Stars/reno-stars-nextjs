### Summary

Investigated and fixed landing page performance issues based on user feedback. Split the monolithic HomePage into 11 smaller components with code-splitting. ### SEO Improvements

#### Removed Homepage Suspense Skeleton

**Problem:** The Suspense wrapper with `HomePageSkeleton` fallback caused the initial HTML to contain skeleton content instead of actual page content, harming SEO (crawlers index skeleton instead of real content).

**Fix:** Removed Suspense wrapper from `app/[locale]/page.tsx`. The server now fully renders the homepage with all data before sending HTML. Data fetching still uses `Promise.all` for parallelization.

#### FAQ Schema for Service Pages

Added bilingual FAQ content to translation files and FAQSchema to service detail pages:

- **`messages/en.json`** ‚Äî Added `faq` section with 3 Q&A pairs for each of 6 service types
- **`messages/zh.json`** ‚Äî Added corresponding Chinese translations
- **`app/[locale]/services/[service-slug]/page.tsx`** ‚Äî Imports FAQSchema, builds FAQ array from translations, renders in page

Each service type has unique, relevant FAQs:
- Kitchen: timeline, cost, permits
- Bathroom: timeline, usage during reno, waterproofing
- Whole-house: living arrangements, project management, scope
- Basement: permits, moisture, bathroom addition
- Cabinet: refacing vs replacing, timeline, eligibility
- Commercial: business hours, codes, tenant improvements

#### New Structured Data Schemas

Added 2 new JSON-LD schema components in `components/structured-data/`:

| Component | Schema Type | Used On |
|-----------|-------------|---------|
| `ProjectSchema.tsx` | CreativeWork | Project detail pages |
| `LocalBusinessAreaSchema.tsx` | HomeAndConstructionBusiness | Area pages |

**ProjectSchema** includes:

**LocalBusinessAreaSchema** includes:
- Company info scoped to specific area
- `areaServed` as City type
- `hasOfferCatalog` with all services

Updated `components/structured-data/index.ts` to export new schemas.

#### Meta Description Truncation

Added SEO-optimized truncation utility and applied to key pages:

**`lib/utils.ts`** ‚Äî New `truncateMetaDescription()` function:
```typescript
export function truncateMetaDescription(text: string, maxLength: number = 155): string
```
- Truncates at word boundaries (doesn't cut mid-word)
- Appends ellipsis when truncated
- Default 155 chars (optimal for SERP display)

Applied to:
- `app/[locale]/blog/[slug]/page.tsx`
- `app/[locale]/projects/[slug]/page.tsx`
- `app/[locale]/services/[service-slug]/page.tsx`
