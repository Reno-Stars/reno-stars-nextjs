## Session 3: Fix "Browse by Category" Bug

Category cards on the projects page only showed "Whole House" — all other categories (Kitchen, Bathroom, Commercial) were missing.

### Root Cause
Category filtering compared localized category names (`p.category === c[locale]`), but project `category_en` values in the DB were inconsistent with service titles used as category names (e.g., project has `"Kitchen"` but service title is `"Kitchen Renovation"`). Only "Whole House Renovation" matched exactly because that string happened to be identical in both the project `category_en` and the service title.

### Fix: Match by `service_type` instead of localized name
- **Files:** `lib/db/queries.ts`, `components/pages/ProjectsPage.tsx`
- `getCategoriesLocalized()` now returns `{ serviceType: string; en: string; zh: string }[]` — the `serviceType` field is the service slug (e.g., `"kitchen"`, `"bathroom"`) matching the project's `service_type` column
- All category matching in `ProjectsPage` now compares `project.service_type === category.serviceType` instead of `project.category === category[locale]`
- Extracted `matchesCategory(project, serviceType)` helper to DRY the matching logic (used in dropdown filter, browse-by-category cards, and card count)
- Special handling for `'whole-house'`: matches `p.isSiteProject || p.service_type === 'whole-house'` (sites don't have `service_type`)
- `activeCategory` state now stores service type slugs (e.g., `"kitchen"`) instead of English names (e.g., `"Kitchen Renovation"`)
