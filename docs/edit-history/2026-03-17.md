# 2026-03-17

## Session 1: Website Audit Fixes

Fixed issues identified from a website audit of the production deployment.

### 1. Improved 404 page description text
- **Files:** `messages/en.json`, `messages/zh.json`, `app/[locale]/not-found.tsx`
- Added `common.notFoundDescription` key with proper 404 copy instead of generic "An error occurred"
- EN: "The page you're looking for doesn't exist or has been moved."
- ZH: "您访问的页面不存在或已被移动。"
- Updated `not-found.tsx` to use `t('notFoundDescription')` instead of `t('error')`

### 2. Added Google Maps embed to contact page
- **Files:** `components/pages/ContactPage.tsx`, `proxy.ts`
- Added a map iframe between "Service Areas" and "Why Us" sections in the info column
- Uses free Google Maps embed (no API key): query-based URL with business name + address
- URL extracted to `MAP_EMBED_URL` constant at top of file for maintainability
- Styled with neumorphic card (`neu(4)`) and `overflow-hidden` for rounded corners
- `loading="lazy"`, `allowFullScreen`, accessible `title` attribute via `t('footer.mapLocation')`
- Added `frame-src 'self' https://www.google.com` to CSP in `proxy.ts` (was blocked by `default-src 'self'` fallback)

### 3. Fixed sitemap `lastModified` for static pages
- **Files:** `app/sitemap.ts`
- Added `STATIC_LAST_MODIFIED = '2026-03-17'` constant
- Static pages now use fixed date instead of `new Date().toISOString()` on every deploy
- Prevents search engines from thinking all static pages are "updated" on each deployment
- Dynamic pages (projects, blogs, sites) still use their DB `updatedAt` timestamps

### 4. Improved contact form accessibility
- **Files:** `components/ContactForm.tsx`
- Added `id="form-error"` to the inline error alert div
- Added `aria-required="true"` on required inputs (name, phone, message)
- Added `aria-describedby="form-error"` on all inputs/textarea when an error is shown
- `aria-describedby` is conditional — only set when `errorMessage` is truthy (avoids referencing a non-existent element)
