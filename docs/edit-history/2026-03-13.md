# 2026-03-13 — Chunked Multipart Upload for Batch Upload

## Summary

Replaced the single-body server proxy upload with S3 multipart chunked upload to support large ZIP files (up to 1 GB) without hitting server memory or timeout limits. Previous approach buffered the entire file in server memory; new approach uploads 10 MB chunks via `CreateMultipartUpload` → `UploadPart` × N → `CompleteMultipartUpload`. Added per-chunk retry, input validation, and centralized the `temp/batch/` S3 key prefix.

## Changes

### Server Route — Multipart Upload Endpoints

- **`app/admin/(dashboard)/batch-upload/api/[jobId]/upload/route.ts`** — Rewrote from single `PUT` (buffered full file) to three endpoints:
  - `POST`: Initialize S3 multipart upload, returns `uploadId`. Verifies job exists and is pending.
  - `PUT`: Upload a single chunk (~10 MB). Query params: `uploadId`, `partNumber`. No per-chunk DB verification (avoids ~100 queries for 1 GB file). Returns `etag`.
  - `PATCH`: Complete multipart upload. Validates each part entry has `partNumber` (number ≥ 1) and `etag` (non-empty string). Best-effort `AbortMultipartUpload` on failure. Body parsed once before try/catch so `uploadId` is available for abort.
- Added shared `verifyPendingJob()` helper used by POST and PATCH handlers.

### Client — Chunked Upload with Retry

- **`app/admin/(dashboard)/batch-upload/BatchUploadClient.tsx`** — Replaced XHR-based single upload with chunked `fetch` loop:
  - File split into `CHUNK_SIZE` (10 MB) parts. S3/R2 minimum is 5 MB for non-final parts.
  - Per-chunk retry: `CHUNK_MAX_RETRIES = 3` with 1s/2s backoff.
  - Progress = chunks completed / total chunks.
  - Uses `AbortController` for cancellation (replaced `xhrRef` with `abortRef`).
  - Constants moved to module scope.

### Shared S3 Key Helper

- **`lib/batch/types.ts`** — Added `batchZipKey(jobId)` function returning `temp/batch/${jobId}.zip`.
- **`lib/batch/batch-processor.ts`** — Replaced two hardcoded `temp/batch/${jobId}.zip` strings with `batchZipKey()` import.
- **`app/admin/(dashboard)/batch-upload/api/[jobId]/upload/route.ts`** — Uses `batchZipKey()` for all S3 operations.

- **`docs/admin-components.md`** — Updated batch upload description from presigned URL to chunked multipart proxy.
